## 什么是增强式学习？🤔

到目前为止，我们讲解的几乎都是监督学习（Supervised Learning）。例如，要训练一个图像分类器，你不仅需要告诉机器输入是什么，还需要告诉它应该输出什么样的正确答案（标签）。本课程目前涉及的技术大多基于监督学习的方法。

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/1f31c4df6cb250346b06f5262024a7bf_7.png)

![](images/e0fc58741541.png)

即使是在讲解自监督学习（Self-supervised Learning）时，我们也采用了类似监督学习的方法，只是标签不需要人工标注，而是可以自动生成。在讲解自动编码器（Auto-Encoder）时，我们虽然称其为无监督方法，但实际上我们仍然有一个“标签”，只是这个标签不需要耗费人力来产生。

![](images/2200cd658dcd.png)

![](images/5c71d27ba132.png)

![](images/797dc752dd37.png)

然而，RL是另一个面向的问题。在RL中，当我们给机器一个输入时，我们并不知道最佳的输出应该是什么。举例来说，假设你要让机器学习下围棋。用监督学习的方法似乎也可以做：你告诉机器下一步应该落子的位置。但问题是，下一步最佳的落子位置究竟在哪里？哪一步是“神之一手”？人类可能根本不知道。

![](images/265fea64ffb1.png)

![](images/3fd6231feb42.png)

当然，你可以让机器阅读大量职业棋手的棋谱，棋谱中人类下的下一步可能是一个很好的答案，但它是不是最好的答案呢？我们并不知道。在这种你不知道正确答案是什么的情况下，往往就是RL可以派上用场的时候。

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/1f31c4df6cb250346b06f5262024a7bf_21.png)

所以，当你发现收集有标注的数据很困难，或者正确答案连人类也不知道的时候，也许就可以考虑使用RL。但是，RL在学习时，机器也并非一无所知。我们虽然不知道正确的答案是什么，但机器会知道什么是“好”，什么是“不好”。机器会与环境互动，得到一个叫做“奖励”（Reward）的信号。通过与环境互动，机器可以知道什么样的输出是好的，什么样的输出是不好的，从而学习出一个模型。

## RL与机器学习的三个步骤 🔄

![](images/665fe5e07b49.png)

![](images/c20b224a8bb1.png)

在本学期课程的一开始，我们就提到机器学习就是三个步骤。RL作为机器学习的一种，也遵循一模一样的三个步骤。

**第一步：定义一个带有未知参数的函数。**  

在RL中，这个函数就是我们的“智能体”（Actor）。智能体就是一个函数，它接收环境（Environment）给出的“观察”（Observation）作为输入，然后输出一个“动作”（Action）。这个动作会影响环境，环境随后会给出新的观察和奖励。我们的目标就是找到一个函数（智能体），使得它在于环境互动的过程中，获得的总奖励（Total Reward）最大。

**第二步：定义一个损失函数（Loss Function）。**  

在RL中，我们的目标是最大化总奖励 ( R )。我们可以将负的总奖励 ( -R ) 定义为损失函数，这样我们的目标就变成了最小化这个损失，与机器学习框架保持一致。

**第三步：进行优化（Optimization）。**  

我们需要找到一组参数（即智能体网络中的权重），使得损失函数最小（即总奖励最大）。然而，RL的优化比一般的监督学习更复杂，因为环境通常是一个具有随机性的“黑盒子”，我们无法直接使用梯度下降法。

## 具体例子：太空侵略者游戏 🎮

为了更好地理解，我们以经典的“太空侵略者”（Space Invader）游戏为例。

![](images/2e4bc674f464.png)

- **环境**：游戏主机，控制外星人并生成游戏画面。
- **智能体（Actor）**：玩家控制的太空船。
- **观察（Observation）**：游戏画面（像素）。
- **动作（Action）**：三个可能的行为：左移、右移、开火。
- **奖励（Reward）**：游戏分数。例如，击中外星人得5分，其他动作得0分。

智能体的目标是学习一个策略，通过选择一系列动作，使得整场游戏获得的总分数（总奖励）最高。

## RL的难点与策略梯度（Policy Gradient）简介 🧩

上一节我们介绍了RL的基本框架和优化目标。本节中我们来看看RL的难点以及一个基础的解决方案。

RL的优化之所以困难，主要因为：

1. **智能体的输出具有随机性**：即使输入相同的观察，智能体每次采取的动作也可能不同（例如根据概率采样）。
2. **环境是黑盒子**：我们不知道环境内部的具体运作机制，它可能也具有随机性。
3. **奖励函数可能很稀疏**：例如在下围棋时，只有游戏结束时才获得奖励（赢+1，输-1），中间步骤的奖励均为0。

因此，我们不能直接使用标准的反向传播来训练智能体网络。一个常用的方法是**策略梯度（Policy Gradient）**。

![](images/5df5808ab175.png)

我们可以将训练智能体在特定状态下采取特定动作的问题，类比成一个分类问题：

- **输入**：观察 ( s )（如图像）。
- **标签**：我们希望智能体采取的动作 ( \hat{a} )。
- **损失**：智能体输出动作分布与目标动作之间的交叉熵（Cross Entropy）。

如果我们希望智能体在状态 ( s ) 下采取动作 ( \hat{a} )，就最小化交叉熵损失。如果我们希望智能体**避免**采取某个动作，就最大化交叉熵损失（即最小化负的交叉熵）。

![](images/b8779b28e35c.png)

更进一步，我们可以为每个“状态-动作对” ( (s, \hat{a}) ) 赋予一个权重 ( A \，表示我们**多希望**智能体执行这个动作。( A ) 可以是正数（希望执行），也可以是负数（希望避免），其绝对值大小代表期望的强烈程度。此时的损失函数可以定义为：  

`Loss = -Σ (A_n * CrossEntropy(a_n, \hat{a}_n))`  

通过最小化这个损失，我们可以训练智能体按照我们的期望程度来采取行动。

然而，这里引出了两个关键问题：

1. 这些训练数据（即哪些状态应该/不应该采取哪些动作）从哪里来？
2. 如何确定每个动作的期望权重 ( A ) ？

![](images/b7f87c4d8a6a.png)

这两个问题正是策略梯度等RL算法的核心。我们将在后续课程中深入探讨。

## 总结 📝

本节课中我们一起学习了增强式学习（RL）的基础概念：

1. RL适用于那些没有明确标准答案，但可以通过与环境互动获得“奖励”信号来学习的问题。
2. RL与机器学习一样，包含三个步骤：定义函数、定义损失、优化参数。
3. 我们通过“太空侵略者”游戏例子，理解了智能体、环境、观察、动作和奖励这些核心概念。
4. RL的优化难点在于环境的随机性和黑盒特性。
5. 我们初步了解了策略梯度（Policy Gradient）的思想，即将RL的部分训练过程转化为一个加权的分类问题，其中如何生成训练数据和确定权重是关键。

![](images/7aadb335efd2.png)
