## 作业任务描述

本次作业的目标是实现对抗攻击。攻击类型是非目标性攻击，采用的范数是L∞范数，扰动上限ε为8。我们将使用FGSM和I-FGSM这两种基础攻击算法。

上一节我们介绍了作业的整体目标，本节中我们来看看具体的攻击场景。这是一个经典的黑盒攻击场景：你需要定义并攻击自己的代理模型，而助教的模型对你来说是未知的。你的目标是生成对抗样本，这些样本不仅要能欺骗你自己的模型，还要能成功转移到助教的模型上。

下图是一个经典的对抗攻击示例：原始图片被模型正确分类为“熊猫”，但添加了精心构造的噪声后，模型会以高置信度将其错误分类为“长臂猿”。

![](images/b4457ca032aa.png)

## 数据与模型

我们统一使用CIFAR-10数据集。但请注意，助教用于评估的黑盒模型是未知的。

你需要选择一些代理模型来生成对抗图片。这些对抗图片需要能成功欺骗你自己的模型，并尽可能迁移到黑盒模型上。

以下是本次作业需要完成的核心部分：

- 选择代理模型进行攻击，以实现向黑盒模型的迁移。
- 实现非目标性对抗攻击方法。基础代码已提供FGSM和I-FGSM。
- 实现更高级的方法以获得更高分数（Bonus部分）：
  
  MI-FGSM（动量迭代快速梯度符号法）
  使用Diverse Input（DI）方法增强攻击的迁移性。
  实现集成攻击，同时攻击多个代理模型。

## 攻击算法详解

上一节我们列出了需要实现的方法，本节我们来详细看看这些算法的原理。

![](images/38eee35bae13.png)

### FGSM（快速梯度符号法）

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/1d0a5215a858013fed6c4c3d36c38ca3_10.png)

这是一个经典的攻击方法。其目标是生成一个对抗样本 `x_adv`，在扰动不超过 `ε` 的前提下，使模型的损失函数值增大。

公式如下：  

`x_adv = x + ε * sign(∇x J(θ, x, y))`  

其中，`sign` 函数取梯度的符号（+1或-1），`∇x J` 是损失函数相对于输入 `x` 的梯度。

### I-FGSM（迭代快速梯度符号法）

这是FGSM的迭代版本。与单步的FGSM不同，I-FGSM通过多次小步幅的迭代来生成对抗样本。

其更新步骤如下：  

`x_adv^{t+1} = Clip_{x, ε} { x_adv^t + α * sign(∇x J(θ, x_adv^t, y)) }`  

其中，`t` 是迭代次数，`α` 是每一步的步长（通常 `α = ε / T`，`T`为总迭代数）。`Clip` 函数确保每次更新后的 `x_adv` 始终在原始样本 `x` 的 `ε` 邻域内。

### MI-FGSM（动量迭代快速梯度符号法）

为了帮助对抗样本逃离局部最优，MI-FGSM在迭代过程中引入了动量项，类似于优化中的动量梯度下降。

![](images/a676cdba8a30.png)

其核心是在计算梯度方向时，累积之前迭代的梯度信息：  

`g_{t+1} = μ * g_t + (∇x J(θ, x_adv^t, y)) / ||∇x J(θ, x_adv^t, y)||_1`  

`x_adv^{t+1} = Clip_{x, ε} { x_adv^t + α * sign(g_{t+1}) }`  

其中，`g_t` 是累积的动量，`μ` 是衰减因子。

![](images/012a5e9222df.png)

### Diverse Input (DI) 方法

为了让攻击对不同模型和变换更具鲁棒性（即提高迁移性），DI方法在每次迭代时以一定概率对输入图像进行随机变换。

变换主要包括随机调整大小和随机填充。以概率 `p` 对输入应用变换 `T(·)`，然后计算梯度：  

`x_adv^{t+1} = Clip_{x, ε} { x_adv^t + α * sign(∇x J(θ, T(x_adv^t), y)) }`  

这种方法可以防止对抗样本过拟合到特定的代理模型上。

### 集成攻击 (Ensemble Attack)

![](images/47285e5aad36.png)

![](images/da7fcf06fa9e.png)

![](images/a9f74619c6b8.png)

![](images/ebeda0b770e2.png)

同时攻击多个代理模型可以生成更具通用性的对抗样本。其思想是优化一个同时最大化多个模型损失的目标函数。

![](images/a1f87aa28135.png)

一个简单的方法是对多个模型的损失进行加权求和：  

`J_ensemble(θ, x, y) = Σ_i w_i * J_i(θ_i, x, y)`  

其中，`w_i` 是第 `i` 个模型的权重，`Σ_i w_i = 1`。然后使用这个集成损失来计算梯度并进行攻击。选择哪些模型进行集成也是一门学问，相关论文提供了指导。

以下是不同等级的评分基线：

- **Baseline**： 实现基础的FGSM攻击即可通过。
- **Medium**： 需要实现集成攻击（Ensemble Attack）。
- **Strong**： 需要实现集成攻击，并合理选择代理模型，或使用MI-FGSM等方法。
- **Boss**： 需要实现集成攻击，精心选择代理模型，并结合MI-FGSM和DI等方法。

1. **环境设置与数据加载**： 已配置好PyTorch和CIFAR-10数据加载。**请勿修改**数据预处理部分（如归一化参数和`ε`的设置），否则可能导致扰动超出限制。
2. **攻击算法框架**：
  
  `fgsm`： 实现了单步FGSM攻击。
  `i_fgsm`： 实现了迭代I-FGSM攻击，步长`α`可调。
  `mi_fgsm`： **需要你完成**动量项的计算部分。
  `ensemble attack`： **需要你完成**多个模型损失的加权求和逻辑。
3. **对抗样本生成与保存**：
  
  调用 `generate_adv_examples` 函数，传入模型、数据、攻击算法和损失函数，即可生成对抗样本。
  使用 `create_adv_dataset` 函数将生成的对抗样本保存为指定格式的图片文件。
  最后将整个文件夹压缩成 `.tgz` 格式提交到Kaggle。
4. **防御实验部分**： 提供了使用JPEG压缩对抗样本并观察类别变化的代码框架，你需要运行并记录结果以回答报告问题。
