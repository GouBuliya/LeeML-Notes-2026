在本节课中，我们将学习机器学习任务的核心攻略。这份攻略将帮助你理解如何诊断和解决模型在训练与测试中遇到的各种问题，从而在作业中取得更好的成绩。我们将从检查训练损失开始，逐步分析模型偏差、优化问题、过拟合以及数据不匹配等情况，并提供相应的解决策略。

## 作业任务概览

![](images/04cc98357f1f.png)

首先，我们来回顾一下课程作业的基本形式。后续的多个作业格式都大同小异。

以下是各个作业的简要描述：

- **作业一**：你已经看过，是预测任务的示例。
- **作业二**：语音辨识（简化版）。输入 `X` 是一小段声音信号，输出 `Y` 是预测该信号对应的音标（如KK音标）。
- **作业三**：影像辨识。输入 `X` 是一张图片，输出 `Y` 是判断图片中的物体。
- **作业四**：语者辨认。输入 `X` 是一段声音信号，输出 `Y` 是判断说话人的身份。
- **作业五**：机器翻译。输入 `X` 是一种语言的句子，输出 `Y` 是另一种语言的翻译句子。

训练过程遵循三个标准步骤：

1. 定义一个带有未知参数 **θ** 的函数：`f_θ(x)`
2. 定义一个损失函数 **L(θ)**，用于评估参数的好坏。
3. 解决优化问题，找到一组参数 **θ***，使得损失最小：`θ* = argmin_θ L(θ)`

![](images/be9556d996ad.png)

找到 **θ*** 后，将其应用于测试数据，得到预测结果。

![](images/a3262bdb3f0c.png)

## 攻略总览：从训练损失开始

当你在 Kaggle 上的结果不理想时，第一步不是检查测试损失，而是**检查训练数据上的损失**。因为模型首先需要在训练数据上学好。

如果训练损失很大，说明模型在训练集上就没学好。这通常有两个可能的原因：**模型偏差** 或 **优化问题**。

![](images/6de99b4ee015.png)

## 情况一：训练损失大

### 可能原因 1：模型偏差

![](images/633fcd4355f5.png)

![](images/f0bb3d19e54e.png)

模型偏差是指你的模型过于简单，其函数集合中根本不包含能够使损失足够低的函数。

![](images/cde5a85ae69b.png)

**公式化描述**：假设你的模型参数 **θ** 所有可能取值构成的函数集合为 **F**。如果最优函数 **f***（能使损失最小）不在 **F** 中，即 `f* ∉ F`，那么即使找到集合内的最优参数 **θ***，损失仍然很高。

![](images/20a5160290ba.png)

**解决方案**：重新设计模型，增加其灵活性。

- 增加输入特征。
- 使用更大的模型（例如，增加网络层数或神经元数量）。
- 使用深度学习模型。

![](images/07ee8fb89e23.png)

### 可能原因 2：优化问题

优化问题是指，尽管你的模型函数集合中确实存在低损失的函数，但优化算法（如梯度下降）未能找到它。

![](images/b89e987cea7a.png)

**概念描述**：模型集合 **F** 中包含好的函数（“针在海里”），但梯度下降给出的 **θ*** 对应的函数损失不够低（“没能把针捞起来”）。

![](images/d9fb8448252a.png)

![](images/8a6ec60608d4.png)

**如何判断是模型偏差还是优化问题？**  

建议通过比较不同复杂度的模型来判断。  

例如，一个56层的网络在训练集上的损失比20层的网络还要高。由于56层网络的容量更大（它至少可以模拟20层网络的行为），理论上应能得到更低或相等的训练损失。如果结果更差，则很可能是优化出了问题。

![](images/850617ffc676.png)

![](images/6a2d5fd53e5e.png)

**解决方案**：

1. 对于新问题，可以先训练一些较浅或较简单的模型（如线性模型），以了解“简单模型能做到多好”。
2. 当更复杂、弹性更大的模型无法达到或超越简单模型的训练损失时，就表明存在优化问题。解决优化问题的方法将在后续课程中介绍。

上一节我们介绍了训练损失大的两种原因及其解法。接下来，我们假设训练损失已经降得很低，看看测试阶段会遇到什么问题。

## 情况二：训练损失小，测试损失大

当训练损失很小，但测试损失很大时，你很可能遇到了**过拟合**问题。

![](images/814f6bfb06c6.png)

**过拟合的直观解释**：模型能力过强，完美拟合了训练数据中的细节甚至噪声，但在训练数据未覆盖的区域“自由发挥”，导致在未见过的测试数据上表现糟糕。

![](images/4927c05f4cbb.png)

**解决过拟合的方法**：

1. **增加训练数据**：这是最有效的方法。更多数据可以限制模型的“自由发挥”，迫使其学习更通用的模式。
  
  *注意*：在课程作业中，通常不允许自行收集额外数据。
  **数据增强**：基于对问题的理解，人工创造合理的训练数据变体。例如，在图像识别中，对图像进行左右翻转、裁剪缩放等。
2. **给模型增加限制**：降低模型的灵活性，使其无法过度拟合训练数据。
  
  使用更少的参数。
  使用更少的特征。
  让模型参数共享权重（如卷积神经网络 CNN）。
  使用早停、正则化、Dropout 等技术（后续课程会详述）。

![](images/e15647e33890.png)

### 模型复杂度的权衡

模型的复杂度需要谨慎选择。复杂度与损失的关系通常如下图所示：

- **训练损失**：随着模型复杂度增加而持续下降。
- **测试损失**：随着模型复杂度增加先下降后上升。下降阶段是模型学习有效规律，上升阶段则是发生了过拟合。

我们的目标是选择一个复杂度适中的模型，使得测试损失最小。

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/2837a3ea8f9bd845888ccd91600e7331_51.png)

## 如何选择模型？验证集的使用

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/2837a3ea8f9bd845888ccd91600e7331_53.png)

你可能会想：直接拿不同模型在 Kaggle 的 Public Testing Set 上测试，选分数最高的不就行了？**强烈不建议这样做！**

**原因**：这会导致你实际上是在用 Public Testing Set 来调整模型，相当于“偷看”了部分考试答案。这会使模型过度拟合 Public Set，而在最终评判的 Private Testing Set 上表现很可能大幅下滑，即“公开榜排名高，最终榜崩盘”。

**正确做法**：使用**验证集**。

1. 将训练数据划分为两部分：**训练集** 和 **验证集**。
2. 只用训练集来训练模型。
3. 用验证集上的损失（或准确率）来评估和选择模型。
4. 将选出的最佳模型用于 Public/Private Testing Set 进行最终评估。

**为什么这样有效**：验证集是从训练数据中分出的、模型在训练时未见过的一部分，它能较好地模拟模型在真正测试集上的表现，同时避免了“偷看”测试答案。

**进阶技巧：N折交叉验证**  

如果担心单次划分验证集不够稳定或具有偶然性，可以使用 N 折交叉验证。

1. 将训练数据平均分成 N 份。
2. 依次将其中一份作为验证集，其余 N-1 份作为训练集，进行 N 次训练和验证。
3. 计算每个模型在这 N 次验证中的平均表现，选择平均表现最好的模型。
4. 用全部训练数据重新训练该最佳模型，用于测试。

## 特殊情况：数据不匹配

除了过拟合，还有一种情况是**数据不匹配**。即训练数据和测试数据来自不同的分布。

**例子**：用2020年的视频观看数据训练，预测2021年的数据。由于年份间可能存在根本性变化（如疫情影响生活习惯），模型会表现很差。增加2020年的训练数据也无济于事。

![](images/7beb0288f368.png)

![](images/a7f67808c377.png)

**与过拟合的区别**：过拟合可以通过增加**同分布**的训练数据缓解；数据不匹配则需要获取与测试集分布更接近的训练数据，或使用领域自适应等技术。

![](images/b3e250aea6ef.png)

**如何判断**：需要对数据来源和背景有深入理解。课程中的作业11就是专门针对数据不匹配问题设计的。

## 总结

![](images/255f6f6237eb.png)

![](images/6c7c72aab54e.png)

![](images/8f257689581f.png)

![](images/c62833eedf4b.png)

本节课我们一起学习了机器学习任务的系统化攻略：

1. **诊断训练损失**：若训练损失大，需判断是**模型偏差**（增加模型复杂度）还是**优化问题**（改进优化算法）。
2. **诊断测试损失**：若训练损失小、测试损失大，可能是**过拟合**。解决方法包括增加数据、数据增强、给模型增加限制。
3. **模型选择**：使用**验证集**或**N折交叉验证**来选择模型，避免使用 Public Testing Set 做选择导致在 Private Set 上表现不佳。
4. **注意特殊情况**：留意**数据不匹配**问题，这需要不同的解决思路。
