在本节课中，我们将学习自注意力机制的各种变体。自注意力是处理序列数据（如句子、音频或图像）的核心技术，但其计算量会随着序列长度的平方增长，这在处理长序列时成为瓶颈。因此，研究者们提出了多种方法来加速或改进标准的自注意力机制。我们将逐一介绍这些方法，了解它们的设计思路、优缺点以及适用场景。

## 自注意力机制回顾

![](images/2c74ed1a9d57.png)

上一节我们介绍了自注意力机制的基本概念。本节中，我们首先简要回顾其工作原理和计算难点。

自注意力机制用于处理一个长度为 ( N ) 的输入序列。该序列可以是句子中的词汇、音频帧或图像像素。对于序列中的每个位置，模型会生成一个查询向量 ( q )、一个键向量 ( k ) 和一个值向量 ( v )。

接下来，所有查询向量和键向量会进行两两之间的点积运算，总共进行 ( N^2 ) 次计算，从而得到一个 ( N \times N ) 的注意力矩阵。然后，根据这个矩阵对值向量进行加权求和，得到每个位置的输出。

其核心计算过程可以表示为以下公式：

![](images/57789d2b0dae.png)

[  

\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V  

]

![](images/a831992d6f3c.png)

![](images/d2fbaad56549.png)

其中，( Q )、( K )、( V ) 分别是由所有查询、键、值向量组成的矩阵。

**计算难点**：注意力矩阵的计算复杂度为 ( O(N^2) )。当输入序列长度 ( N ) 非常大时（例如处理高分辨率图像，( N ) 可达数万），计算量会变得难以承受。因此，加速自注意力计算的关键在于减少这个 ( N \times N ) 矩阵的计算负担。

## 基于先验知识的稀疏注意力

![](images/76ceefd628a6.png)

一种直观的加速思路是：我们可能不需要计算注意力矩阵中所有位置的值。根据对特定任务的理解，我们可以预先设定某些位置的注意力权重为零，从而避免计算。以下是几种基于此思路的方法。

### 局部注意力 (Local/Truncated Attention)

在某些任务中，一个位置可能只需要关注其邻近位置的信息。

![](images/13cddc24d34f.png)

**具体做法**：为每个位置设定一个固定的关注窗口（例如，只关注左右各 ( w ) 个位置）。在注意力矩阵中，只有窗口内的位置需要计算权重，窗口外的位置直接设为零。

![](images/528c4fb6c5a6.png)

```python
# 伪代码示意：局部注意力掩码
# 假设序列长度 N=5，窗口大小 w=1
mask = [
    [1, 1, 0, 0, 0],  # 位置1关注1,2
    [1, 1, 1, 0, 0],  # 位置2关注1,2,3
    [0, 1, 1, 1, 0],  # 位置3关注2,3,4
    [0, 0, 1, 1, 1],  # 位置4关注3,4,5
    [0, 0, 0, 1, 1],  # 位置5关注4,5
]
# 只计算mask中为1的位置的注意力
```

![](images/7c4f83f3c8d3.png)

**过渡**：局部注意力虽然能加速，但限制了信息的远程交互。为了看到更广的范围，研究者提出了步进注意力。

![](images/0225031c6aa7.png)

### 步进注意力 (Strided Attention)

![](images/ec0553b8295a.png)

为了扩大感受野，可以让每个位置跳过固定间隔去关注其他位置。

![](images/a4c51bc2dac0.png)

**具体做法**：例如，每个位置关注距离为 ( s, 2s, 3s, ... ) 的位置。在注意力矩阵中，会形成一种带状模式。

```python
# 伪代码示意：步进注意力掩码 (stride=2)
# 位置1关注1,3,5；位置2关注2,4；以此类推
```

![](images/6160e19269cc.png)

**优缺点**：这种方法比局部注意力能看到更远的依赖，但模式仍然是固定的。

### 全局注意力 (Global Attention)

![](images/c612aa591104.png)

有时我们需要一些“特殊”的令牌来汇总整个序列的全局信息。

**具体做法**：在输入序列中加入一个或多个特殊的全局令牌（例如，[CLS] 标记）。这些全局令牌会与序列中的所有其他令牌相互计算注意力（即它们关注所有人，也被所有人关注）。而其他普通令牌之间则不直接计算注意力。

![](images/38c0801f208e.png)

```python
# 伪代码示意：全局注意力掩码（假设前两个位置是全局令牌）
mask = [
    [1, 1, 1, 1, 1],  # 全局令牌1关注所有位置
    [1, 1, 1, 1, 1],  # 全局令牌2关注所有位置
    [1, 1, 0, 0, 0],  # 普通令牌3只关注全局令牌
    [1, 1, 0, 0, 0],  # 普通令牌4只关注全局令牌
    [1, 1, 0, 0, 0],  # 普通令牌5只关注全局令牌
]
```

**比喻**：全局令牌就像是社区中的“村长”，每个人都认识村长，村长也认识每个人，信息通过村长来传递，但村民之间可能互不相识。

![](images/6f03961c389f.png)

**过渡**：那么，我们应该选择哪种注意力模式呢？一个聪明的答案是：全都要。

### 组合多种模式：Longformer 与 Big Bird

在实际模型中，我们可以利用自注意力机制的多头特性，让不同的头采用不同的注意力模式。

- **Longformer**：结合了局部注意力、步进注意力和全局注意力。
- **Big Bird**：在 Longformer 的基础上，额外增加了**随机注意力**，即随机选择一些令牌对让它们直接计算注意力。这有助于信息在图中更高效地传播。

## 基于学习的稀疏注意力

![](images/66a442056e9c.png)

上一节介绍的方法依赖于人工设计的模式。我们能否让模型自己学习哪些位置的注意力是重要的呢？以下是几种基于学习的稀疏化方法。

![](images/5318a081f1d1.png)

### 聚类注意力 (Clustering-based Attention)

![](images/651e073176ef.png)

核心思想：如果查询向量和键向量在语义上不相似，它们的注意力权重可能很小。我们可以快速地将相似的向量聚类，只计算同一簇内向量之间的注意力。

**具体步骤**：

1. 对所有查询和键向量进行快速聚类（例如使用哈希或近似方法）。
2. 对于每个查询，只计算它与同一簇内的键的注意力权重，与其他簇的键的注意力直接设为零。

**代表工作**：Reformer, Routing Transformer。

![](images/4a9815583589.png)

**过渡**：聚类方法仍然基于“相似度”的假设。有没有更彻底的方法，完全让一个神经网络来学习注意力掩码呢？

![](images/718c3ca12f4a.png)

### 学习生成注意力掩码：Sinkhorn Sorting Network

![](images/28c0f260ffd7.png)

这种方法旨在用一个可学习的神经网络来直接生成一个二值的注意力掩码矩阵，决定哪些位置需要计算注意力。

**具体做法**：

1. 输入序列通过一个网络，产生一个连续的 ( N \times N ) 矩阵。
2. 通过 Sinkhorn 归一化等可微操作，将这个连续矩阵转化为一个二值（0/1）的掩码矩阵。
3. 训练时，这个生成掩码的网络与主网络一起通过反向传播进行优化。

![](images/17afe53f2f25.png)

**核心挑战与技巧**：为了让生成过程可微，论文采用了特殊的技术。此外，为了降低生成掩码本身的计算成本，通常会让一组输入向量共享同一个掩码模式（即生成一个分辨率较低的掩码，再上采样到 ( N \times N )）。

![](images/3294b23bd1fc.png)

![](images/34ad67e2c227.png)

**过渡**：Sinkhorn Network 先产生掩码，再计算注意力。一个更极端的想法是：为什么不直接让网络输出注意力权重呢？这就引出了 Synthesizer。

![](images/4fa8d104c698.png)

## 低秩近似与核化方法

我们之前一直试图生成一个稀疏的 ( N \times N ) 矩阵。但有没有可能，这个矩阵本身的信息就是冗余的，我们可以用更小的矩阵来近似它呢？

![](images/31fbbff5a189.png)

### 选取代表性键值：Informer, Compressed Attention

![](images/4b13942d0eae.png)

研究发现，注意力矩阵通常是低秩的，许多列是冗余的。因此，我们可以只选取一小部分（( K ) 个）具有代表性的键和值来进行计算，从而将计算复杂度从 ( O(N^2) ) 降至 ( O(NK) )。

![](images/5666429d2e9f.png)

**如何选取代表性键**：

- **Compressed Attention**：使用卷积神经网络（CNN）对键序列进行下采样，输出更短的序列作为代表性键。
- **Linformer**：将键矩阵 ( K )（( D \times N )）乘以一个可学习的投影矩阵 ( E )（( N \times K )），得到降维后的代表性键矩阵 ( \tilde{K} = K E )。这相当于对原始 ( N ) 个键向量进行 ( K ) 种不同的线性组合。

![](images/43967ef00a8d.png)

**为什么通常只减少键，不减少查询**？因为输出的序列长度与查询数量相同。对于需要为每个输入位置都产生一个输出的任务（如序列标注），减少查询数量会导致输出序列变短，这可能不符合任务要求。

![](images/d80044b2506a.png)

**过渡**：除了在矩阵元素上做文章，我们还可以从矩阵乘法的计算顺序上优化。

### 改变计算顺序：线性注意力 (Linear Attention)

回顾自注意力的计算：( O = \text{softmax}(QK^T) V )。如果不考虑 softmax，其本质是三个矩阵相乘 ( V \cdot (K^T \cdot Q) )。矩阵乘法的结合律告诉我们，先计算 ( (V \cdot K^T) ) 再乘 ( Q )，结果相同，但计算量可能不同。

![](images/9baa9560f78a.png)

**计算量对比**：

- 原始顺序 ( (QK^T)V )：复杂度约为 ( O(D N^2) )。
- 改变顺序 ( V(K^T Q) )：先计算 ( \phi = V K^T )，得到一个 ( D_v \times D ) 的矩阵，再计算 ( O = \phi Q )，复杂度约为 ( O(D_v D N) )。

当序列长度 ( N ) 远大于向量维度 ( D ) 时，改变顺序可以显著降低计算量，从 ( N^2 ) 级别降至 ( N ) 级别。

**核心技巧：核函数 (Kernel Function)**  

但标准自注意力包含 softmax 非线性操作。为了应用上述优化，需要将 `exp(q·k)` 分解为两个函数的内积：`exp(q·k) = φ(q)·φ(k)`。找到这样的特征映射 ( φ ) 是关键。

![](images/3d05cd92f5b6.png)

**代表工作**：Performer, Linear Transformer, Random Feature Attention 等，它们的区别主要在于所使用的核函数 ( φ ) 不同。

![](images/087ec074e2a6.png)

**理解新视角**：在这种方法下，可以理解为先基于所有键值对，计算出一组固定的“模板”向量。然后，每个位置的输出就是其查询向量与这组模板进行线性组合的结果。

## 对注意力机制的再思考

![](images/78ecb93ce125.png)

![](images/53892575da4e.png)

前面的方法都在改进或加速标准的 Q-K-V 注意力框架。但有些研究开始质疑这一框架本身。

### 合成注意力 (Synthesizer)

Synthesizer 提出：注意力权重矩阵不一定非要由查询和键计算得出。

![](images/ac6b8521f600.png)

![](images/0d2026d0b150.png)

**两种变体**：

1. **Dense Synthesizer**：注意力权重矩阵直接作为模型参数，是固定的，不随输入变化。
2. **Random Synthesizer**：注意力权重由一个前馈网络根据输入直接生成。

**意义**：即使使用固定的注意力矩阵（所有输入共享），模型性能下降也不大。这促使我们重新思考注意力的本质价值：它可能更多是提供了一种强大的、结构化的信息混合方式，而非精确的“相关性”计算。

**过渡**：如果注意力权重可以固定甚至随机，那么我们能否完全抛弃注意力机制呢？

![](images/ab4c0ceb0245.png)

![](images/3eb10ae81921.png)

### 无注意力模型 (Attention-Free Models)

![](images/93ecb4015cf2.png)

这是一个新兴的研究方向，旨在探索不使用注意力机制来处理序列的方法。例如，完全使用多层感知机（MLP）或状态空间模型（如 S4, Mamba）来建模长序列依赖。这开启了“后注意力时代”的探索，是另一个广阔的故事。

## 总结与性能对比 🏁

![](images/e5fb8eb98f82.png)

本节课我们一起学习了多种自注意力机制的变体，它们的目标都是在保持或小幅牺牲性能的前提下，显著提升计算效率，以处理更长的序列。

![](images/0fa5fd1e6da1.png)

下表总结了部分代表性方法的核心思路：

| 方法类别 | 代表模型 | 核心思想 | 优点 | 缺点/挑战 |
| --- | --- | --- | --- | --- |
| **基于先验的稀疏化** | Longformer, Big Bird | 人工设计局部、步进、全局等注意力模式。 | 直观，可解释性强。 | 模式固定，可能不是最优。 |
| **基于学习的稀疏化** | Reformer, Sinkhorn Net | 通过聚类或网络学习重要的注意力位置。 | 自适应数据。 | 学习过程可能增加开销。 |
| **低秩与近似** | Informer, Linformer | 选取代表性键值对，用更小的矩阵近似。 | 理论上有保证，效率提升明显。 | 选取代表的方法影响效果。 |
| **核化与线性注意力** | Performer, Linear Trans. | 改变计算顺序，利用核技巧避免显式 NxN 矩阵。 | 计算复杂度降至 O(N)。 | 需要设计合适的核函数。 |
| **结构再思考** | Synthesizer | 注意力权重不由QK计算，甚至作为固定参数。 | 挑战传统认知，计算简单。 | 性能上限和理论解释待研究。 |

根据在 **Long Range Arena** 基准测试上的表现，我们可以观察到：

- **Big Bird**（组合模式）和 **Longformer** 性能接近原版 Transformer，速度略有提升。
- **Reformer**（聚类）和 **Sinkhorn**（学习掩码）速度较快，但性能有一定下降。
- **Performer** 和 **Linear Transformer**（线性注意力）速度提升非常显著，是处理超长序列的有力工具，性能下降在可接受范围内。
- **Local Attention** 速度最快，但性能下降也最多。

![](images/e2ffea4a6e1d.png)

没有一种方法在所有指标上都最优。在实际应用中，需要根据任务对序列长度、计算资源和性能的要求来权衡选择。自注意力机制的创新仍在继续，理解这些变体有助于我们为不同场景选择最合适的工具，并启发新的思路。
