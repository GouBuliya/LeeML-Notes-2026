# 贡献指南 / Contributing

感谢你愿意为这份笔记贡献力量 ❤️
Thank you for considering a contribution ❤️

## 你能做什么 / What you can do

| 类型 | 难度 | 我们最需要的 |
|---|---|---|
| 🐛 错别字 / 排版 | ⭐ | 直接 PR 即可 |
| 🖼️ 配图破损 / 替换更清晰版本 | ⭐ | 提 Issue 报告章节号 |
| ✍️ 章节内容补充 / 注解 | ⭐⭐ | 先开 Issue 沟通方向 |
| 🔗 配套代码 / Colab 链接 | ⭐⭐ | 直接 PR,引用作者署名 |
| 🌐 翻译(英 / 日 / 繁中) | ⭐⭐⭐ | 先开 Issue,商量协作分工 |

## PR 流程 / Pull Request Flow

```bash
# 1. Fork 本仓库到你自己的 GitHub
# 2. clone 下来
git clone https://github.com/YOUR-FORK/LeeML-Notes-2026.git
cd LeeML-Notes-2026

# 3. 创建分支
git checkout -b fix/typo-chapter-36

# 4. 修改后提交
git add .
git commit -m "fix: 修正第 36 讲第 5 段 self-attention 公式排版"

# 5. push 并发起 PR
git push origin fix/typo-chapter-36
```

## 提交信息规范 / Commit Convention

简单遵循 conventional commits:

```
fix: 修复 XX 章节的错别字
docs: 补充 XX 配图说明
feat: 新增 XX 章节注解
chore: 调整目录结构
```

## 注意事项 / Notes

- **不要随意搬运他人内容**:如果你引用了别处的解读 / 图,请在 PR 里注明出处。
- **保持原意**:这份笔记的核心是李宏毅老师的课程,补充内容应当**辅助理解**,而非替换原意。
- **不要加广告 / 引流**:任何带有商业目的(微信群引流、付费课程链接等)的 PR 会被直接 close。

## 行为准则 / Code of Conduct

讨论保持友善,**针对内容不针对人**。
违反基本网络礼仪的言行会被 ban。
