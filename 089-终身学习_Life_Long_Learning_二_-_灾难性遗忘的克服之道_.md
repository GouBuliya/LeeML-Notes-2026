# 89：终身学习 (Life Long Learning) (二) - 灾难性遗忘的克服之道 🧠

![](images/790ebacb5a36.png)

在本节课中，我们将要学习克服机器学习中“灾难性遗忘”问题的几种核心方法。灾难性遗忘是指模型在学习新任务时，会严重遗忘旧任务知识的现象。我们将深入探讨三种主要的解决思路，并理解其背后的原理与实现方式。

上一节我们介绍了终身学习的概念和灾难性遗忘问题，本节中我们来看看具体的解决方法。

## 选择性突触可塑性 (Selective Synaptic Plasticity) 🔒

第一个解法叫做**选择性突触可塑性**。从字面上看，这个方法的目标是：只让神经网络中的一部分神经元或连接具有“可塑性”（即可被改变），而另一部分连接则被“固化”，其数值在后续学习中不再改变。

![](images/790ebacb5a36.png)

这类方法也称为**基于正则化 (Regularization-based)** 的方法，是终身学习领域中发展最完整的研究方向。我们将花较多时间讲解它。

### 灾难性遗忘为何发生？

![](images/f641b8b69f41.png)

为了理解解法，我们先分析问题产生的原因。假设有两个任务：任务一和任务二。为简化，假设模型只有两个参数：`θ1` 和 `θ2`。

- 左右两张图分别代表任务一和任务二的损失函数（即误差曲面）。
- 颜色越偏蓝色代表损失越小，越偏白色代表损失越大。
- 我们从随机初始参数 `θ0` 开始，通过梯度下降在任务一上训练，得到参数 `θB`。
- 接着，我们将 `θB` 作为初始值，在任务二的误差曲面上继续训练，得到最终参数 `θ*`。

此时，`θ*` 在任务二上表现好（处于蓝色低损失区），但若将其拿回任务一上测试，会发现表现很差，因为 `θ*` 的位置对任务一而言损失很高。这就是遗忘。

### 核心解决思路

解决思路是：**每个参数对旧任务的重要性不同**。对于旧任务重要的参数，我们在学习新任务时应尽量保持其不变；只去更新那些对旧任务不重要的参数。

具体做法是：为每个参数 `θi` 配备一个“守卫”值 `bi`。`bi` 的大小代表了该参数对旧任务的重要性，`bi` 越大，意味着该参数越重要，越不希望被改变。

![](images/ba9662c9b651.png)

我们通过修改损失函数来实现这一约束。新的损失函数 `L'` 为：

`L' = L(θ) + Σ_i [ bi * (θi - θi^B)^2 ]`

其中：

- `L(θ)` 是新任务原本的损失。
- `θi^B` 是从旧任务学得的参数值。
- 第二项是正则化项，它惩罚新参数 `θi` 与旧参数 `θi^B` 的偏离程度。
- `bi` 控制惩罚的强度。`bi` 大则强制参数不变，`bi` 小则允许参数改变。

如果所有 `bi = 0`，则退化为普通训练，会发生灾难性遗忘。  

如果所有 `bi` 都设得非常大，则新参数被强制接近旧参数，这会导致模型无法学习新任务，这种现象称为 **顽固性 (Intransigence)**。

因此，关键在于如何为每个参数设定合适的 `bi` 值。

![](images/47d693697ef2.png)

### 如何设定 `bi` 的值？

`bi` 是人为设定的，其核心思想是：**评估每个参数对旧任务损失的影响程度**。

![](images/6c72e9e0c5f5.png)

- 如果一个参数（如 `θ1`）的微小变动对旧任务的损失影响很小，则说明它对旧任务不重要，可以赋予较小的 `b1`，允许它在学习新任务时自由更新。
- 如果一个参数（如 `θ2`）的微小变动会显著增大旧任务的损失，则说明它很重要，应赋予较大的 `b2`，限制它在学习新任务时的变化。

通过这种方式，在训练新任务时，梯度更新的方向就会被引导至那些对旧任务不重要的参数方向上，从而在学会新任务的同时，最大程度地保护旧任务的知识。

![](images/6c72e9e0c5f5.png)

![](images/2df66df170d7.png)

### 实验结果

下图展示了一个经典方法 **EWC** 的实验结果。图中横轴是依序训练任务A、B、C的过程，纵轴是各任务的正确率。

![](images/2df66df170d7.png)

- **蓝色线 (bi=0)**：发生明显的灾难性遗忘。学完B后，A的正确率下降；学完C后，A和B的正确率都下降。
- **绿色线 (所有bi=1)**：虽然缓解了遗忘（A的正确率保持较好），但产生了顽固性，导致新任务B和C学不好（正确率上升缓慢或不高）。
- **红色线 (选择性设置bi)**：在防止遗忘（A的正确率保持好）和学习新任务（B和C的正确率较高）之间取得了最佳平衡。

文献中存在多种计算 `bi` 的具体方法（如 EWC, SI, MAS, RWalk, SCP等），它们各有特点，是作业中探讨的重点。

## 基于梯度方向修正的方法：Gradient Episodic Memory (GEM) 📐

GEM 方法不在参数值上施加约束，而是在**梯度更新方向**上施加约束。

其做法是：

1. 在新任务（任务二）上计算梯度 `G`。
2. 在更新参数前，回到旧任务（任务一）上计算其梯度方向 `G_old`（蓝色箭头）。
3. 如果新任务的梯度 `G` 与旧任务的梯度 `G_old` 方向不一致（内积为负），则对 `G` 进行修正，得到一个新的更新方向 `G‘`。
4. `G‘` 需要满足：与 `G_old` 的内积非负（不损害旧任务），同时与原始 `G` 的差异尽可能小（尽可能完成新任务的学习）。

![](images/2851dd824610.png)

![](images/2851dd824610.png)

这种方法的一个潜在问题是：计算旧任务的梯度 `G_old` 需要用到旧任务的部分数据。这与终身学习希望不存储旧数据的初衷略有违背。但 GEM 通常只需存储少量数据即可，如果其存储开销小于存储整个旧模型和 `bi` 值的开销，则仍是一个可行的方案。

## 动态分配神经资源 (Additional Neural Resource Allocation) 🧩

这种思路的核心是：**为不同的任务分配或使用不同的网络参数**，避免直接修改旧任务所用的参数。

以下是两种代表性方法：

### 渐进式网络 (Progressive Neural Network)

![](images/827d94ad0cd9.png)

![](images/edc12da22e7c.png)

- **做法**：冻结为任务一训练好的网络。当学习任务二时，**新建一个网络**，该网络可以接收任务一网络的隐藏层输出作为额外输入。任务二只训练这个新增网络的参数。
- **优点**：完全避免了灾难性遗忘。
- **缺点**：每学一个新任务，模型就会增长一部分。随着任务增多，模型会变得非常庞大。

![](images/0081e44eb996.png)

![](images/db34f8590887.png)

![](images/f053455ff4a3.png)

![](images/f053455ff4a3.png)

### 打包网络 (PackNet) 与 压缩-打包-增长 (CPG)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/412d58837b4e5169d761f2cd837aba3b_31.png)

- **PackNet做法**：预先初始化一个较大的网络。学习每个新任务时，只使用和更新网络中分配给该任务的**一部分参数**，其他参数被“掩码”冻结。
- **优点**：模型总参数量固定，不会无限增长。
- **CPG做法**：结合了渐进和打包的思想，模型既可以增长新参数，每个任务又只使用部分参数，是一种更灵活的策略。

![](images/7d6a75509575.png)

![](images/7d6a75509575.png)

## 记忆回放 (Memory Replay) 🔄

![](images/666185e5c9f5.png)

这是一个非常直观的方法：既然把新旧数据混合训练可以避免遗忘，但又不能存储旧数据，那么我们可以训练一个**生成模型 (Generative Model)** 来“回忆”旧数据。

- **做法**：
  
  在学习任务一时，同时训练一个分类器和一个能够生成类似任务一数据的生成器。
  学习任务二时，不仅使用任务二的真实数据，还使用任务一的生成器产生的“伪数据”来一起训练分类器。
  如此循环，每学完一个任务，就更新生成器使其能生成所有已学任务的数据。
- **评价**：该方法需要额外训练和维护生成模型，会占用一定资源。但在实践中，这种方法往往能取得接近“多任务学习”（同时看到所有数据）的性能上限，效果非常显著。

## 扩展讨论与总结 🎯

### 任务顺序的重要性

任务的**学习顺序**对终身学习的效果有显著影响。在某些顺序下，灾难性遗忘问题可能更严重；而在另一些顺序下，问题可能自然缓解。因此，在严谨的研究中，通常会测试所有可能的任务排列顺序并取平均性能。

![](images/84e9b282e94a.png)

![](images/a28222b6caa4.png)

### 更复杂的场景

我们讨论的场景假设所有任务的结构（如类别数）相同。实际上，终身学习还有更复杂的场景，例如：

- **增量分类**：新任务会引入新的类别。
- **领域增量学习**：任务的数据分布发生变化，但类别空间可能不变。
- **任务增量学习**：本节课主要讨论的类型，模型需要明确知道当前是哪个任务。

![](images/cfbd94ed929a.png)

这些场景有相应的研究方法（如 LwF, iCaRL等），是更前沿的探索方向。

本节课中我们一起学习了克服机器学习中灾难性遗忘的三大类方法：

1. **选择性突触可塑性**：通过给参数施加重要性加权正则化，保护重要参数。
2. **梯度 episodic memory**：通过修正梯度更新方向，使其不损害旧任务。
3. **动态资源分配与记忆回放**：通过分配独立参数或生成伪数据，物理上隔离或模拟旧任务数据。

![](images/35af595db826.png)

每种方法都有其优缺点和适用场景，理解其核心思想是设计或应用终身学习算法的关键。终身学习是一个活跃的研究领域，仍有诸多挑战等待解决。
