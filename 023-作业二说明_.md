## 音素分类任务详解

上一节我们介绍了作业的整体目标，本节中我们来看看任务的具体细节。

音素分类，即Phoneme Classification，其目标是为语音片段标注音标。例如，对于单词“machine”，我们可以将其声音切分为多个片段，并为每个片段识别出其对应的音素，如 /m/, /ə/, /ʃ/, /iː/, /n/。

![](images/602cff908f1f.png)

直接使用原始的波形数据进行分类非常困难，因为数据量庞大且格式不适合。因此，标准的处理流程是：

1. 将音频切分为帧。
2. 将每一帧转换为特征向量。

### 帧与特征提取

我们通常以25毫秒为一个窗口（或称为一帧）进行切分，然后每10毫秒滑动一次窗口以切分下一帧。这样相邻帧之间会有重叠，以确保不会遗漏重要的声音过渡部分。

切分后，我们需要将每一帧音频转换为更适合分类的特征。常用的特征是**MFCC**。

![](images/bdacf23f86b0.png)

**MFCC** 是梅尔频率倒谱系数的缩写。其计算过程可以简化为以下步骤：

1. 对原始波形进行傅里叶变换，得到频谱图。
2. 通过一组梅尔滤波器组过滤频谱图，模拟人耳对不同频率的感知。
3. 对滤波后的结果进行进一步处理，最终得到39维的MFCC特征向量。

因此，最终用于分类的数据格式是：一个形状为 `(T, 39)` 的张量，其中 `T` 是帧的数量，`39` 是MFCC特征的维度。

### 上下文拼接

![](images/020c3cb894ea.png)

由于单个25毫秒的帧包含的信息有限，为了帮助模型更好地判断中间帧的音素，我们会使用上下文信息。具体做法是，对于目标帧，我们将其前后各5帧的特征向量拼接起来，形成一个更大的输入向量。

经过这种拼接操作后，输入模型的向量维度变为 `11 * 39 = 429` 维。模型将基于这429维的信息来判断中间帧的音素标签。

## 数据集说明

上一节我们了解了任务的处理流程，本节中我们来看看提供给你们的实际数据。

我们使用的是 **TIMIT** 数据集的一个子集。你们无需自行处理原始音频，我们已经完成了MFCC特征提取。

![](images/e45d35e656db.png)

以下是数据集的目录结构：

![](images/4e2d21899087.png)

```
根目录/
├── train_split.txt    # 训练集元数据（文件名列表）
├── test_split.txt     # 测试集元数据（文件名列表）
├── train_labels.txt   # 训练集标签
└── features/          # 特征文件夹
    ├── xxx.pt
    ├── yyy.pt
    └── ...
```

![](images/ba2ec4d784fd.png)

- `train_split.txt` 和 `test_split.txt` 指明了哪些文件属于训练集或测试集。
- `train_labels.txt` 包含了训练集中每个音频文件对应的所有音素标签序列。
- `features/` 文件夹中的 `.pt` 文件是使用PyTorch保存的张量。每个文件对应一个音频文件转换后的MFCC特征，形状为 `(T, 39)`。

## 示例代码导读

![](images/336105cbc9e7.png)

最后，我们简要浏览一下提供的示例代码，帮助大家上手。

![](images/c919a7ab889f.png)

![](images/0c10385d434b.png)

代码主要结构如下：

1. **数据加载**：读取 `train_split.txt`, `train_labels.txt` 和 `features/` 下的文件。
2. **数据预处理**：包含一个 `concat_feature` 函数，用于实现帧的上下文拼接。
  
  该函数接收一个特征张量 `(T, 39)` 和一个奇数 `N`（例如11，表示拼接前后各5帧）。
  它返回拼接后的新张量，形状为 `(T, 39*N)`。**请确保处理后帧的数量 `T` 没有错误减少。**
3. **数据集构建**：将处理后的所有帧特征和标签整合成最终的训练集 `(X, Y)`。
4. **模型定义**：一个简单的全连接神经网络。
5. **训练循环**：标准的PyTorch训练流程。

![](images/7645e4a80d26.png)

![](images/d361420ad209.png)

- `concat_feature` 函数中拼接的帧数 `N`。
- 模型架构的层数和每层的神经元数量。
- 训练超参数（学习率、迭代次数等）。
