## 任务概述 📋

作业三的目标是使用课程中教授过的CNN来解决图像分类问题。任务涉及实物分类，数据集将实物分为11个类别。具体数据量如下：

- **训练集**：9866张照片
- **验证集**：3430张照片
- **测试集**：3347张照片

以下是本次作业的规则，请特别注意：

- 禁止使用数据集的真实标签进行训练。
- 禁止使用任何额外的数据来训练模型。
- **禁止使用预训练模型**。因为预训练模型能力过强，会使得任务变得过于简单。同时，在竞赛期间也不允许尝试使用预训练模型来规避规则。
- 允许使用任何公开的代码或套件。
- 由于允许使用公开代码，因此在竞赛期间**禁止公开自己的代码或模型**。
- 在使用公开套件时，需注意检查其是否默认使用预训练模型，大部分套件都是，需要将其修改为不使用预训练模型的版本。

### 2. 数据增强

![](images/6da5c6085e35.png)

如果模型总是看到相同的照片，很容易导致过拟合。数据增强通过对训练图片进行随机变换（如旋转、裁剪、颜色抖动等），使模型学习到更通用的特征表示。

![](images/eeb03ec11a5b.png)

我们可以使用`torchvision.transforms`模块来实现数据增强。在代码中，通常需要修改`train_tfm`这个函数来加入数据增强操作。

![](images/b682eb2431ee.png)

一个进阶的数据增强方法是**MixUp**。其思想很简单：将两张图片及其标签进行线性组合，生成新的训练样本。

**MixUp公式**：

- 新图像 = λ * 图像A + (1 - λ) * 图像B
- 新标签 = [λ * 标签A的概率， (1 - λ) * 标签B的概率]

实现MixUp需要修改以下部分：

1. 修改`Dataset`的`__getitem__`函数，使其能读取并组合两张图片。
2. 修改标签返回格式，使其能返回两个标签及其对应的混合权重（λ）。
3. 由于标准的交叉熵损失函数不支持多标签输入，因此需要自己实现MixUp对应的损失函数。

### 3. 测试时数据增强

通常，在测试时我们会对图片进行标准化处理以获得清晰的输入。但事实上，如果在测试时也使用多种训练阶段的数据增强方法生成多张图片，然后对它们的预测结果进行集成（如投票或平均），可能会获得更好的性能。

![](images/ce266b93fa65.png)

实现方法是：将测试时的数据增强机制改为使用训练时的增强方法，并进行多次预测。由于原始的测试变换（标准化）通常效果更好，我们可以为其赋予更高的权重。

例如，用训练增强生成5张图片的逻辑输出进行平均，测试增强生成1张图片的逻辑输出单独计算，最终的预测逻辑可以是测试输出的权重更高一些。

### 4. 交叉验证

![](images/822a5e29f1fa.png)

交叉验证将训练集和验证集重新划分为不同的子集，从而可以训练多个看到不同数据分布的模型，集成后通常能获得更好的效果。

![](images/f52667b9598d.png)

在代码实现上，需要将训练集和验证集的图片路径合并，然后自己重新划分训练集和验证集。即使不打算训练多个模型，也建议重新划分数据集，因为原始数据集中验证集比例（约1/4）可能过高，减少验证集数量、增加训练集数量有助于提升模型性能。

### 5. 模型集成

以下是两种最常见的集成方法：

- **逻辑输出平均**：将多个模型对同一张图片输出的逻辑值（logits）直接取平均，然后取argmax得到最终类别。优点是结果连续，不易出现平局。缺点是需要保存所有模型的完整输出。
- **投票法**：用多个模型分别预测，然后对预测的类别进行投票。优点是实现简单，只需要保存最终的预测类别文件（CSV）。缺点是需要处理票数相同的情况。

本次作业强烈建议使用Kaggle平台，因为它具有以下优势：GPU内存为16GB；数据I/O速度更快；更容易保存和加载训练中的检查点。

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/93f9d45db95ecfb593479ebbbc924018_50.png)

![](images/198d3ab9b6c2.png)

**注意事项**：Kaggle有每周30小时的GPU使用时间限制，每次任务最长运行时间为12小时。

以下是使用Kaggle的核心步骤：

1. 在竞赛页面点击“New Notebook”创建笔记本。
2. 通过“Add Data”添加本次作业的数据集（搜索`ml2023spring homework3`）。
3. 在“Settings”的“Accelerator”选项中选择GPU。
4. 可以逐格执行代码进行调试，也可以提交整个笔记本在后台运行。
5. 运行结束后，在“Output”标签页的“Data”中下载生成的`submission.csv`文件。
6. 如果需要从检查点继续训练，可以将输出的模型文件保存为新的数据集，然后在新的笔记本中导入该数据集并加载模型继续训练。

**时间管理技巧**：

- Kaggle会在每周六重置GPU时间，并根据你上周的使用情况分配新额度。建议尽早开始使用。
- 后台运行任务时，即使时间额度用完，当前任务也不会被中断。可以利用这一点在额度快用完时提交一个长时间运行的任务来“延长”可用时间。
- 最多可以同时运行两个后台任务，可以利用这一点同时训练两个不同随机种子或不同模型结构的任务，以节省时间。

## 其他实用技巧与报告要求 📝

上一节我们介绍了Kaggle的使用，本节中我们来看看其他提升性能的技巧和作业报告的要求。

**提升性能的技巧**：

- **数据增强是必须的**，否则模型很容易过拟合。多样化的增强方法能为测试时增强提供更多样本。
- 如果从提供的示例模型结构开始修改，**不要害怕将模型变大**。许多预定义的模型结构都很大。实验发现，具有**下采样**结构的模型通常表现更好。
- 可以尝试其他技巧，如分类任务中的**标签平滑**、**Focal Loss**，优化中的**Dropout**、**梯度累积**（当模型很大导致batch size很小时有用）、**批归一化**和**图像归一化**。

**作业报告要求**：

1. **实现训练数据增强**：将你编写好的`train_tfm`函数代码粘贴到GradeScope对应问题中。该函数必须能对同一张图片产生至少五种不同的输出。
2. **实现残差连接**：在提供的`Residual_Network`类中，修改`forward`函数以实现残差连接。残差连接的思想是将某一层的输入与该层的输出相加，公式可表示为：`输出 = F(x) + x`，这有助于缓解深层网络中的梯度消失问题。将修改后的类代码粘贴到GradeScope即可。
