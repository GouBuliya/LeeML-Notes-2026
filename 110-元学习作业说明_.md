在本课程中，我们将学习如何完成一个基于元学习（Meta-Learning）的少样本分类（Few-Shot Classification）作业。我们将使用Omniglot数据集，并实现一个五路单样本（5-way 1-shot）分类任务。

---

## 概述

本次作业的目标是实现一个元学习算法，用于解决少样本分类问题。具体任务是：给定一个包含5个不同类别（每个类别1张示例图片）的支持集（Support Set），模型需要正确分类查询集（Query Set）中的5张图片。

我们将使用Omniglot手写字符数据集。该数据集包含多种不同书写体系的字符，非常适合用于少样本学习任务。

### 五路单样本分类任务

在每次任务中：

1. 随机选择5个不同的字符类别。
2. 为每个类别采样：
  
  **1张图片**作为支持集（用于模型快速适应）。
  **1张图片**作为查询集（用于测试模型分类能力）。
3. 模型的目标是：根据支持集的5张图片，正确分类查询集中的5张图片。

![](images/f32d57762594.png)

### 数据加载

以下是数据加载的核心逻辑，助教已提供相关代码：

```python
# 伪代码：任务采样
def sample_task(dataset, ways=5, shots=1):
    # 1. 随机选择 `ways` 个类别
    selected_classes = random.sample(dataset.classes, ways)
    support_set = []
    query_set = []
    # 2. 为每个类别采样 `shots` 张支持图片和查询图片
    for cls in selected_classes:
        support_images = sample_images_from_class(cls, shots)
        query_images = sample_images_from_class(cls, shots)
        support_set.append(support_images)
        query_set.append(query_images)
    return support_set, query_set
```

**训练/验证集**：数据加载器会实时从Background Set中采样任务。  

**测试集**：助教已预先采样好640个任务对（每个任务包含5张支持图片和5张查询图片），文件结构已固定，请勿修改加载顺序。

## 基准模型与实现目标

了解了任务格式后，我们来看看本次作业需要实现的几个不同难度的基准模型。

以下是需要完成的四个基准模型，难度递增：

1. **简单基准（Simple Baseline）**：使用基于迁移学习（Transfer Learning）的分类模型。在训练时进行常规分类训练，在测试时根据支持集图片调整分类器。助教已提供实现代码。
2. **中等基准（Medium Baseline）**：实现**一阶元学习算法（First-Order MAML）**。需要完成代码中的 `inner loop` 和 `outer loop` 更新部分。
3. **强大基准（Strong Baseline）**：在中等基准上，实现完整的MAML或其它元学习算法（如Reptile）。
4. **老板基准（Boss Baseline）**：在强大基准的基础上，加入**任务增强（Task Augmentation）**。例如，通过对支持集图片进行旋转、裁剪等变换，创造出新的、相关的任务来提升模型泛化能力。

各基准模型在Kaggle上的预期准确率大致为：

- Simple Baseline: ~0.6
- Medium Baseline: ~0.7
- Strong Baseline: ~0.9
- Boss Baseline: ~0.95
