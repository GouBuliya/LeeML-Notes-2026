## 生成模型的目标 🎯

![](images/5aaa4be34aa2.png)

假设我们要生成图像。我们用 **X** 代表一张图像，它是一个高维空间中的点。为了简化，我们假设 **X** 是二维空间中的一个点。

我们要生成的目标图像有一个固定的分布，我们称之为 **P_data(x)**。这意味着在整个图像空间中，只有一小部分区域采样出的图像看起来像人脸（例如蓝色区域），而其他区域采样出的图像则不像。

我们的目标是让机器找出这个 **P_data(x)** 分布。然而，我们并不知道这个分布的具体数学形式。我们只能从中采样（例如，从数据库中获得图像），但无法写出其公式。

---

## 最大似然估计（MLE）回顾 🔄

![](images/6c687b6d1d1f.png)

![](images/d26a67d801a8.png)

在GAN出现之前，生成模型通常使用最大似然估计。

![](images/a442166e1302.png)

![](images/a11fc7723447.png)

- 我们有一个未知的真实数据分布 **P_data(x)**。
- 我们定义一个由参数 **θ** 控制的模型分布 **P_G(x; θ)**（例如，一个高斯混合模型）。
- 我们的目标是调整参数 **θ**，使得 **P_G(x; θ)** 尽可能接近 **P_data(x)**。

具体做法是：

1. 从 **P_data(x)** 中采样出数据 **{x¹, x², ..., xⁿ}**。
2. 对于每一采样数据 **xⁱ**，计算它从模型分布 **P_G** 中产生的似然（likelihood）**P_G(xⁱ; θ)**。
3. 最大化所有采样数据的**总似然**。这等价于寻找参数 **θ*** 来最大化似然函数 **L**：
  **公式：θ* = argmax_θ Π_i P_G(xⁱ; θ)**

![](images/cec06a42d2f9.png)

实际上，**最大化似然等价于最小化 KL 散度（Kullback–Leibler Divergence）**。KL散度用于衡量两个概率分布之间的差异。

![](images/9c50c3a6b0b0.png)

**公式：Maximize L ⇔ Minimize KL( P_data(x) || P_G(x) )**

![](images/7c803666ce6c.png)

然而，如果 **P_G** 是一个复杂的模型（如神经网络），我们通常无法直接计算其似然 **P_G(x)**，也就难以直接使用最大似然估计。

![](images/92d24d3c7fb2.png)

## GAN的解决方案：网络作为分布 🕸️

在GAN中，生成器 **G** 本身就是一个神经网络，它定义了一个复杂的概率分布 **P_G**。

![](images/1932712c1563.png)

- **生成器 G**：输入一个从简单先验分布（如正态分布）中采样的随机向量 **z**，输出一个样本 **x**，即 **x = G(z)**。
- **分布 P_G**：通过生成器 **G** 产生的所有样本 **x** 的集合，就构成了分布 **P_G**。即使输入 **z** 的分布很简单，复杂的生成器网络也能将其映射成非常复杂的输出分布 **P_G**。

![](images/12f45c76cf29.png)

![](images/a98f7d5ef62a.png)

我们的目标仍然是让 **P_G** 接近 **P_data**。用优化公式表达，即寻找生成器 **G** 以最小化二者之间的某种散度 **Div**：

![](images/5f012691d8f1.png)

**公式：G* = argmin_G Div( P_G, P_data )**

核心问题在于：**P_data** 和 **P_G** 的表达式我们都不知道，如何计算它们之间的散度呢？

## 判别器的作用：衡量散度 📏

GAN的关键在于使用一个**判别器 D** 来衡量两个分布之间的散度。

虽然我们不知道 **P_data** 和 **P_G** 的具体形式，但我们可以从两者中采样：

- 从 **P_data** 采样：从真实数据集中取图像。
- 从 **P_G** 采样：随机采样 **z**，通过生成器 **G** 得到图像。

![](images/1e6adfe41e93.png)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/f7a31ddb6db81c87f5b86cc36d385be0_47.png)

以下是核心思路：

1. 我们训练一个判别器 **D**，其目标是区分来自 **P_data** 的真实样本和来自 **P_G** 的生成样本。
2. 判别器的训练目标函数 **V(G, D)** 如下：
  **公式：max_D V(D, G) = E_{x~P_data}[log D(x)] + E_{x~P_G}[log(1 - D(x))]**
  这个目标函数的意义是：对于真实样本 **x**，希望 **D(x)** 越大越好（接近1）；对于生成样本 **x**，希望 **D(x)** 越小越好（接近0）。
3. 神奇之处在于：**当判别器 D 被训练到最优时（记为 D*），这个最大的目标函数值 V(G, D*) 与 P_G 和 P_data 之间的 JS 散度（Jensen-Shannon Divergence）有直接关系**。
  直观理解：如果两个分布 **P_data** 和 **P_G** 很接近，判别器很难区分它们，其准确率不会很高，对应的 **V** 值也较小，这意味着散度小。反之，如果两个分布差异很大，判别器能轻易区分，**V** 值就大，意味着散度大。

![](images/288a39f2e27e.png)

数学推导可以证明，最优判别器 **D*** 为：

**公式：D*(x) = P_data(x) / [ P_data(x) + P_G(x) ]**

将其代入目标函数 **V** 后，可以得到：

![](images/3dac0d2f0790.png)

**公式：max_D V(D, G) = V(G, D*) = 2 * JS( P_data || P_G ) - 2log2**

因此，**通过优化判别器 D，我们实际上是在衡量 P_G 和 P_data 之间的 JS 散度**。

## GAN的极小极大化博弈 ⚔️

上一节我们介绍了判别器如何衡量分布间的差异。本节中我们来看看如何利用这个衡量结果来优化生成器。

![](images/e86eba86864a.png)

既然判别器 **D** 的最优值给出了散度 **JS(P_G || P_data)**，那么生成器 **G** 的目标就是最小化这个散度。这就形成了一个**极小极大化（Minimax）博弈**：

![](images/f0292221b762.png)

**公式：G* = argmin_G max_D V(D, G)**

![](images/4c632a792871.png)

![](images/76284158228d.png)

这个公式的含义是：

- **内层 max_D**：固定生成器 **G**，寻找最优判别器 **D*** 来最大化 **V**，以准确衡量当前 **P_G** 与 **P_data** 的 JS 散度。
- **外层 min_G**：固定判别器 **D***，更新生成器 **G** 的参数，以最小化刚才衡量出的 JS 散度。

![](images/74c788ef4625.png)

这就好比生成器 **G** 在努力生成以假乱真的样本，而判别器 **D** 在努力提高鉴别能力。两者交替优化，最终目标是使生成分布 **P_G** 无限逼近真实分布 **P_data**。

## 算法实现与实际问题 🛠️

![](images/f9fcfae3ee9f.png)

在实际训练中，我们无法一次性找到全局最优的 **D*** 和 **G***，而是采用迭代的近似方法：

以下是GAN的基本训练算法步骤：

1. **初始化**生成器 **G** 和判别器 **D** 的参数。
2. **循环迭代**：
  
  **步骤1：训练判别器 D**
  
  从真实数据分布 **P_data** 中采样一批样本。
  从先验分布中采样一批噪声 **z**，通过当前生成器 **G** 生成一批样本。
  将这两批样本分别标记为“真”和“假”，训练判别器 **D** 作为一个二分类器（例如，最小化交叉熵损失）。这相当于在近似求解 **max_D V(D, G)**。通常，判别器会更新多次以达到较好的鉴别能力。
  
  
  **步骤2：训练生成器 G**
  
  从先验分布中采样一批新的噪声 **z**。
  固定判别器 **D**，更新生成器 **G** 的参数，以**最大化判别器对生成样本的判别输出 D(G(z))**。这等价于最小化 **log(1 - D(G(z)))**，但在实践中，为了在训练初期获得更大的梯度，常采用**最大化 log(D(G(z)))** 的替代目标（称为Non-saturating loss）。

**注意**：在训练生成器时，通常只进行少量参数更新（例如一次）。这是因为如果生成器变化太大，之前训练的判别器就不再能准确衡量当前的 JS 散度。

## 理论观点与直观演示的矛盾 🤔

按照Ian Goodfellow的最初理论，当生成分布 **P_G** 完全匹配真实分布 **P_data** 时，最优判别器 **D*** 将对所有输入都输出 0.5（即无法区分），变成一个“失效”的常数函数。

![](images/f237cfa99a7a.png)

![](images/e16c4a3eaf17.png)

![](images/547777d43174.png)

![](images/3495910b6657.png)

![](images/1825daa1deb3.png)

![](images/5c1c71752c49.png)

然而，一些直观演示和后续研究提出了不同的观点。例如，在训练完成后，判别器的输出可能仍然能反映数据分布密度（高密度区域得分低，低密度区域得分高），而非完全平坦。

![](images/b23e57a29afe.png)

这种矛盾揭示了GAN理论仍在发展之中。不同的观点可能引向算法实现上的细微差别（例如，是否在训练判别器时混合使用历史生成器样本），并最终影响训练效果。

## 总结 🎓

本节课中我们一起学习了GAN的核心理论框架：

1. **目标**：让生成器定义的分布 **P_G** 逼近真实数据分布 **P_data**。
2. **挑战**：无法直接计算未知分布之间的散度。
3. **解决方案**：引入判别器 **D**。通过训练 **D** 来最大化目标函数 **V**，其最优值反映了 **P_G** 与 **P_data** 之间的 **JS 散度**。
4. **优化形式**：将问题表述为一个 **极小极大化博弈**：**min_G max_D V(D, G)**。
5. **实际算法**：通过交替、迭代地训练判别器和生成器来近似求解该优化问题。

理解这一理论有助于我们更深入地把握GAN的训练动态和设计原理。下一节课，我们将继续探讨GAN的更多变种和训练技巧。
