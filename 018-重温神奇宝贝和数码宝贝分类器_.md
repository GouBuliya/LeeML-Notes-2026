## 观察数据与定义问题 🔍

![](images/060dba930520.png)

上一节我们介绍了机器学习的基本三步：定义函数、定义损失、优化。本节中我们来看看如何将这些步骤应用到一个具体问题。

![](images/e58c1d6818a3.png)

![](images/a2baf8b9c756.png)

我们的目标是找到一个函数，输入是一只动物（宝可梦或数码宝贝）的图片，输出是它的类别。

首先，我们需要对数据做一些观察。宝可梦和数码宝贝虽然相似，但在画风上有显著差异。数码宝贝的线条通常更复杂，而宝可梦的线条更圆润简单。

![](images/452a9111686d.png)

因此，我们可以设计一个函数，根据图片线条的复杂程度来分类。具体方法是使用边缘检测（edge detection）技术，计算图片中白色像素（即边线）的数量。这个数量代表了线条的复杂程度。

我们定义一个函数 **`e(x)`**，输入图片 `x`，输出其边缘检测后的白色像素数量。

![](images/dac1ef76f95e.png)

## 构建模型与定义损失 ⚙️

基于上述观察，我们可以构建一个简单的分类函数。

我们的模型是一个阈值函数：  

**`f_H(x) = 数码宝贝 if e(x) > H else 宝可梦`**

![](images/8ffe8cc00516.png)

其中，`H` 是我们需要学习的未知参数（阈值）。所有可能的 `H` 值（例如从1到10000）的集合，我们称之为 **`H`**。集合 `H` 的大小 **`|H|`** 被称为模型的**复杂程度**。

![](images/fbc09ee9ca56.png)

接下来需要定义损失函数（Loss Function）。我们有一个数据集 **`D`**，其中包含许多成对的样本 `(x, ŷ)`，`x` 是图片，`ŷ` 是其真实标签（宝可梦或数码宝贝）。

![](images/bcdea703ff76.png)

![](images/2717875c2d6f.png)

我们采用最直观的损失定义：分类错误率。对于数据集 `D` 和阈值 `H`，损失 `L(H, D)` 计算如下：

1. 对于数据集中的每一笔数据 `(x_n, ŷ_n)`，计算小损失 `l`：  
  
  **`l = 1 if f_H(x_n) != ŷ_n else 0`**
2. 对所有数据的 `l` 求平均，得到总损失 `L(H, D)`，即错误率。

![](images/6ba017e883cf.png)

![](images/8b8617cdd90e.png)

## 理想与现实：优化目标 🎯

在理想情况下，如果我们能收集到全宇宙所有的宝可梦和数码宝贝，构成完美数据集 **`D_all`**，我们就能找到全局最优的阈值 **`H*`**，使得损失最小：  

**`H* = argmin_H L(H, D_all)`**

![](images/ee81fce8a4a2.png)

然而现实中，我们只能获得一个从 `D_all` 中采样得到的训练集 **`D_train`**。我们在这个训练集上找到最优阈值 **`H_train`**：  

**`H_train = argmin_H L(H, D_train)`**

核心问题是：**`H_train`** 在全集 `D_all` 上的表现，与真正的理想解 **`H*`** 在 `D_all` 上的表现，差距有多大？我们希望这个差距越小越好。

## 理论推导：何时现实接近理想？ 📖

我们希望现实（`H_train` 在 `D_all` 上的损失）与理想（`H*` 在 `D_all` 上的损失）的差距小于某个小值 δ。

![](images/d3ce40cb2fa1.png)

理论推导表明，如果我们的训练集 `D_train` 满足一个条件，那么这个希望就能实现。这个条件是：**对于集合 `H` 中的每一个可能的阈值 `H`，它在 `D_train` 上计算的损失 `L(H, D_train)`，与在 `D_all` 上计算的损失 `L(H, D_all)` 的差距都很小（小于 ε）**。

![](images/be2379d1c4c2.png)

![](images/e0596f5bda9d.png)

这样的 `D_train` 被称为“具有代表性”的好数据。那么，我们采样到“坏”数据（不具代表性）的几率有多大呢？

![](images/06ecf1eb8639.png)

![](images/18939b5f202a.png)

霍夫丁不等式（Hoeffding‘s Inequality）给出了一个上界。采样到一个“坏”训练集的概率 **P_bad** 满足：  

**`P_bad ≤ 2 * |H| * exp(-2 * N * ε²)`**

其中：

- **`|H|`** 是模型复杂程度（可选参数的数量）。
- **`N`** 是训练集 `D_train` 中的数据量。
- **`ε`** 是我们设定的、要求“接近”的程度。

![](images/8e037fa5e53c.png)

从这个不等式可以得出两个重要结论：

![](images/7ed0b7fc2630.png)

1. **增加训练数据量 `N`**：可以显著降低抽到坏数据的概率。
2. **降低模型复杂度 `|H|`**：同样可以降低抽到坏数据的概率。

以下是让模型表现更稳定的方法：

![](images/e1b73d897f8c.png)

![](images/a07b719a438d.png)

- **收集更多训练数据（增大N）**
- **使用更简单的模型（减小|H|）**

## 模型的权衡：偏差与方差 ⚖️

![](images/ad842fdc900a.png)

![](images/5c609f60448d.png)

![](images/50526768fe47.png)

然而，事情并非那么简单。减少模型复杂度 `|H|` 虽然能让现实更接近理想（减少方差），但也会带来另一个问题：**模型能力受限，可能连理想解 `H*` 本身的表现都很差**。

这导致了机器学习中经典的**偏差-方差权衡（Bias-Variance Trade-off）**：

![](images/bad37f1d6ee8.png)

- **高复杂度模型（大|H|）**：理想解 `H*` 可能很好（低偏差），但 `H_train` 容易远离 `H*`（高方差）。
- **低复杂度模型（小|H|）**：`H_train` 容易接近 `H*`（低方差），但理想解 `H*` 本身可能就很差（高偏差）。

![](images/d3b7da29b033.png)

我们陷入了一个两难境地：既希望理想解足够好（低偏差），又希望现实解能接近理想解（低方差）。

![](images/08198b22af51.png)

## 总结与展望 🚀

本节课中我们一起学习了机器学习原理的核心：

![](images/22b9760c1eed.png)

1. 我们通过一个宝可梦与数码宝贝分类的实例，回顾了机器学习的三个步骤。
2. 我们探讨了“过拟合”的理论根源：从全量数据中采样得到的训练集可能不具有代表性。
3. 理论分析表明，**增加训练数据 `N`** 和**降低模型复杂度 `|H|`** 有助于让模型在训练集和真实分布上的表现更一致（减少过拟合风险）。
4. 我们遇到了机器学习的基本矛盾：**偏差-方差权衡**。简单模型方差小但偏差大，复杂模型偏差小但方差大。

![](images/00575b794576.png)

那么，有没有方法能够打破这个权衡，即找到一个模型，它既能拥有强大的表达能力（低偏差），又能在有限数据下学得稳定（低方差）呢？深度学习的许多技术正是在尝试解决这个问题，这将是后续课程探索的方向。
