## 白箱攻击与黑箱攻击

上一节我们介绍了白箱攻击的基本概念，本节中我们来看看当攻击者不知道模型参数时，情况会如何。

![](images/e62045b83f6b.png)

到目前为止，我们讲解的攻击方法都属于**白箱攻击**。这意味着攻击者需要知道模型的参数才能计算梯度，从而在图像上添加特定的噪声。

```python
# 白箱攻击需要访问模型参数以计算梯度
gradient = compute_gradient(model_parameters, input_image)
adversarial_image = input_image + epsilon * sign(gradient)
```

然而，在实际的线上服务中，模型的参数通常是保密的。这种不知道模型参数情况下的攻击，被称为**黑箱攻击**。你可能会认为，既然不知道参数，攻击线上服务应该很困难。但事实并非如此。

## 如何进行黑箱攻击？

![](images/788db99b3976.png)

![](images/5e8acd2be94e.png)

黑箱攻击的核心思想是使用一个**代理模型**来模拟目标模型的行为。

以下是实现黑箱攻击的步骤：

1. 收集目标模型的输入-输出对（即使不知道参数，也可以向线上服务提交输入并观察输出）。
2. 使用这些数据训练一个代理模型。
3. 对这个代理模型执行白箱攻击，生成对抗样本。
4. 希望这些对抗样本对原始的黑箱目标模型同样有效。

在我们的作业中，你们攻击的Jupyter Notebook上的模型就是一个黑箱。你们用自己的代理模型（例如在CIFAR-10上训练的模型）生成对抗样本，然后提交给黑箱模型进行测试。

![](images/e4283ff3b678.png)

## 黑箱攻击的成功率

![](images/2b6d8c3d98bb.png)

黑箱攻击容易成功吗？答案是肯定的。

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/3027cbafa90220bc691e5200f0ca6193_15.png)

文献中的实验表明，即使代理模型与目标模型结构不同（例如用ResNet-101攻击ResNet-152），黑箱攻击也能显著降低目标模型的准确率。虽然成功率略低于白箱攻击，但模型准确率通常会被降至50%以下。

值得注意的是，黑箱攻击在**非目标攻击**上更容易成功（即只要求模型分类错误，不指定错误类别）。而要实现**目标攻击**（例如将“狗”指定误判为“兔子”）则困难得多。

![](images/9e899b4e2a29.png)

## 提升黑箱攻击成功率：集成攻击

一个有效提升黑箱攻击成功率的技巧是使用**集成攻击**。

![](images/4138b3a6c71e.png)

具体做法是，我们同时使用多个不同的模型作为代理模型，寻找一个能成功欺骗这**多个**模型的对抗样本。实验证明，这样的对抗样本在攻击一个未知的黑箱模型时，成功率会显著提高。这是因为能骗过多个模型的扰动，往往抓住了不同模型之间共有的脆弱性。

![](images/2eb725d41ac6.png)

![](images/e93f5c215c49.png)

## 为什么攻击如此容易成功？

攻击，尤其是黑箱攻击容易成功，这背后的原因仍是一个开放的研究问题。目前一个被广泛讨论的假设是：问题可能出在**数据**本身，而非模型。

有研究通过可视化发现，对于同一张图片（如小丑鱼），在不同模型的高维特征空间中，其“可被正确分类的区域”边界在特定方向上都非常狭窄且相似。一个能让某个模型（如VGG16）分类错误的微小扰动方向，对其他模型也常常有效。

![](images/1765c49c4f8b.png)

这表明，对抗样本可能反映了训练数据中存在的某种固有模式或局限性，而不仅仅是某个特定模型的缺陷。

![](images/9c1e927ddc5b.png)

![](images/3d7089c9bc97.png)

## 更极端的攻击形式

![](images/e95c6b28aadf.png)

![](images/bc36ea2a05e9.png)

![](images/d5137ad8c15c.png)

人类的“恶意”在对抗攻击领域展现得淋漓尽致，以下是一些更极端的攻击形式：

![](images/2f1831dd6941.png)

- **单像素攻击**：仅改变图像中的一个像素，就能导致模型误判。
- **通用对抗攻击**：找到一个固定的噪声模式，将其添加到**任何**图像上，都能使目标模型出错。这非常危险，例如只需将该噪声贴在摄像头上，就可能让整个监控系统失灵。
- **物理世界攻击**：攻击不再局限于数字领域。例如，佩戴一副特制的眼镜可以欺骗人脸识别系统；在停车标志上粘贴特定贴纸，能让自动驾驶汽车将其误认为其他标志（如限速标志）。
- **对抗重编程**：这种攻击“寄生”在一个已训练好的模型上（如ImageNet分类器），通过添加特定的对抗性边框，操控该模型去执行一个它原本未被训练的任务（例如，数图像中方块的数量）。
- **后门攻击**：在模型**训练阶段**发起攻击。攻击者向训练数据中注入一些看起来正常、标签也正确的“毒化”样本。模型训练后表现正常，但只要遇到触发后门的特定模式（如某个特定图案），就会产生指定的错误输出。这警示我们，使用来源不明的公开数据集可能存在风险。

![](images/db823295101c.png)

## 如何防御对抗攻击？

面对如此多样的攻击，我们自然需要研究防御方法。防御大致可分为两类：被动防御和主动防御。

### 被动防御

![](images/e39b7b3ccdd3.png)

被动防御不改变已训练好的模型，而是在模型前增加一个“盾牌”或过滤器，旨在削弱对抗噪声的威力。

![](images/7b60bf213f5f.png)

一个简单而有效的方法是**图像模糊化**。许多对抗噪声是精细且方向特定的，轻微的模糊处理足以破坏其攻击性，同时对原始图像的分类结果影响较小。

![](images/71f0811e70ec.png)

![](images/ee052d1a2f1a.png)

```python
# 简单的模糊化防御示例
import cv2
defended_image = cv2.GaussianBlur(adversarial_image, (5, 5), 0)
```

![](images/9b69bf6abe97.png)

其他被动防御方法还包括图像压缩再解压缩（如JPEG压缩）、或用生成模型对图像进行“净化”重建。

![](images/d7701f365b35.png)

然而，被动防御有一个重大弱点：**一旦防御机制被攻击者知晓，他们就可以在生成对抗样本时，将这一防御步骤也考虑进去，从而绕过防御**。为了增强被动防御，可以引入**随机性**，例如随机改变图像大小、随机填充位置等，让攻击者难以预测确切的防御变换。

![](images/7af766fa6d42.png)

### 主动防御：对抗训练

主动防御的核心思想是在训练阶段就提升模型的鲁棒性，主要方法是**对抗训练**。

其流程如下：

1. 用原始数据训练一个初始模型。
2. 用该模型对训练数据生成对抗样本。
3. 将这些对抗样本与其**正确的原始标签**组成新的训练数据。
4. 将新旧数据合并，重新训练模型。

![](images/2037bd66676e.png)

这个过程可以迭代进行，不断发现并修补模型的漏洞。对抗训练也可以看作一种特殊的数据增强方式，即使没有外部攻击，它也能帮助模型提高泛化能力，减轻过拟合。

```python
# 对抗训练的核心循环概念
for epoch in range(num_epochs):
    # 1. 训练模型
    train_model(model, clean_data)
    # 2. 生成对抗样本
    adversarial_data = generate_attack(model, clean_data)
    # 3. 用对抗样本和干净样本一起训练
    train_model(model, clean_data + adversarial_data)
```

但对抗训练也有局限：它计算开销大，且主要防御训练时所用的特定攻击方法。面对全新的攻击算法，其防御效果可能下降。
