## 知识蒸馏 (Knowledge Distillation) 🧪

知识蒸馏的概念与网络剪枝有相似的精神，但其实现方式不同。

它的核心流程是：先训练一个大型网络，这个网络被称为**教师网络 (Teacher Network)**。然后，我们希望得到一个更小的网络，即**学生网络 (Student Network)**。学生网络并非直接对原始数据学习，而是去学习教师网络的输出。

以下是知识蒸馏的具体步骤：

1. 将训练数据输入教师网络，教师网络会为每个输入生成一个概率分布作为输出。例如，在手写数字识别中，对于一张图片，教师可能输出：数字“1”的概率是0.7，“7”是0.29，“9”是0.01。
2. 将同样的图片输入学生网络，但训练目标不是数据的真实标签，而是教师网络输出的这个概率分布。学生网络的目标是让自己的输出分布尽可能逼近教师的输出分布。

你可能会问：为什么不直接训练一个小网络呢？原因与网络剪枝类似：直接训练的小网络性能，往往不如通过向大网络（教师）学习得到的小网络（学生）性能好。

![](images/86a1bd575dae.png)

知识蒸馏技术并非全新，Hinton在2015年发表的《Distilling the Knowledge in a Neural Network》使其广为人知。但更早的文献（如2013年的《Do Deep Nets Really Need to be Deep?》）已使用了类似概念。

为什么知识蒸馏有效？一个直观的解释是：教师网络为学生提供了额外的“软标签”信息。直接告诉学生“这是一张‘1’的图片”可能很困难，因为‘1’可能与‘7’、‘9’等数字有相似之处。而教师网络会输出“这是‘1’的概率是0.7，是‘7’的概率是0.2”，这揭示了类别间的相似性，使学生网络更容易学习。

![](images/0b36e4ddb5cb.png)

一个神奇的现象是：即使某些类别的训练数据从未给学生网络看过，仅凭教师网络揭示的类别间关系（例如“7”和“1”有点像），学生网络也有可能学会识别该类别。

**教师网络也可以是集成模型 (Ensemble)**。集成是机器学习比赛中常用的技巧，即训练多个模型并将它们的输出进行平均或投票，作为最终结果。虽然集成能提升性能，但计算开销巨大。通过知识蒸馏，我们可以将多个模型集成起来的结果作为教师，让学生网络去学习这个集成输出，从而用一个轻量级的学生网络来逼近集成的性能。

### 知识蒸馏中的技巧：温度 (Temperature)

在知识蒸馏中，常对Softmax函数进行一个小修改，即引入**温度**参数。

首先，回顾标准Softmax公式：  

对于一个神经元的输出向量 **z**，其第 *i* 个类别的概率为：  

**公式**：`p_i = exp(z_i) / sum(exp(z_j)) for j in all classes`

![](images/67a92f6f23be.png)

引入温度 *T* 后的Softmax变为：  

**公式**：`p_i = exp(z_i / T) / sum(exp(z_j / T)) for j in all classes`

其中，*T* 是一个需要调节的超参数。当 *T > 1* 时，它会使原本尖锐的概率分布变得平滑。

为什么要这样做？如果教师网络的输出非常集中（例如某个类别概率为1，其余为0），那么让学生去学习这个输出就和直接学习真实标签没有区别，失去了知识蒸馏的意义。通过设置一个较大的 *T*，可以在不改变类别排序的前提下，让概率分布更平滑，从而让学生网络能更好地学习到类别间的关系信息。

![](images/8a8726834370.png)

## 参数量化 (Parameter Quantization) 🔢

参数量化的目标是：用更少的存储空间来保存网络参数。请注意，在本课程作业中，由于限制的是参数数量而非存储大小，因此此方法可能不直接适用，但仍值得了解。

最简单的量化方法是**降低数值精度**：例如，将参数从32位浮点数（float32）转换为16位浮点数（float16）甚至8位整数（int8），这样存储空间可直接减半，且性能通常下降很小。

更进一步的方法是**权值聚类 (Weight Clustering)**：

1. 对网络中所有权重数值进行聚类（例如分成4类）。
2. 每一类用一个代表值（如该类权重的均值）来表示。
3. 存储时，只需保存一个“聚类中心值表”和每个权重所属的“聚类索引”。如果只有4类，那么每个权重只需2个比特即可表示其索引，极大地压缩了存储空间。

还可以结合**霍夫曼编码 (Huffman Encoding)** 等无损压缩技术，对常见的聚类索引使用更短的码，进一步减少平均存储长度。

![](images/8670811ff3c4.png)

### 二值化权重 (Binary Weights)

参数量化的终极形式之一是**二值化权重**，即每个参数只能是+1或-1。这样每个参数仅需1个比特存储。

![](images/8644dc79f0c7.png)

你可能会认为二值化网络的性能会很差，但事实并非如此。例如，在论文《BinaryConnect》中，作者在MNIST和CIFAR-10数据集上实验发现，二值化网络的错误率有时甚至略低于全精度网络。一个可能的原因是，二值化给模型能力施加了更强的限制，反而有助于防止过拟合。

## 网络架构设计 (Network Architecture Design) 🏗️

通过设计更高效的网络架构，可以直接减少参数量。本节介绍本次作业的主力方法：**深度可分离卷积 (Depthwise Separable Convolution)**。

在讲解它之前，我们先快速回顾标准卷积 (Standard Convolution)：

- 输入特征图有 *I* 个通道。
- 每个卷积核是一个3D张量，其“深度”必须与输入通道数 *I* 相同（例如3x3x*I*）。
- 如果有 *O* 个卷积核，则输出特征图有 *O* 个通道。
- 总参数量 = 卷积核尺寸 (如 K x K) x 输入通道数 *I* x 输出通道数 *O*。

![](images/4887a719877e.png)

深度可分离卷积将标准卷积分解为两个步骤：

**第一步：深度卷积 (Depthwise Convolution)**

- 输入有 *I* 个通道，就使用 *I* 个卷积核。
- 每个卷积核**只负责一个输入通道**，在单个通道上进行卷积。
- 输出特征图通道数仍为 *I*。这一步负责处理**空间特征**（同一通道内像素间的关系），但**忽略了通道间的关系**。

**第二步：逐点卷积 (Pointwise Convolution)**

- 使用 *O* 个大小为 1x1 的卷积核。
- 这些1x1卷积核的作用是：对深度卷积输出的 *I* 个通道进行线性组合，以生成 *O* 个输出通道。
- 这一步专门负责融合**通道间的关系**。

**参数量对比**：

- **深度卷积参数量**：K x K x *I* (每个通道一个KxK核)
- **逐点卷积参数量**：1 x 1 x *I* x *O* (O个1x1xI的核)
- **总参数量**：K x K x *I* + *I* x *O*

与标准卷积参数量 (K x K x *I* x *O*) 的比值约为：`1/O + 1/(K*K)`。由于 *O* 通常较大，比值主要由 `1/(K*K)` 决定。当K=3时，参数量可减少到约1/9；当K=2时，可减少到约1/4。

![](images/08acff07df22.png)

**原理**：深度可分离卷积的本质是一种**低秩近似 (Low-rank Approximation)**。它将一个全连接层（可视为大矩阵）分解为两个更薄矩阵的乘积，从而在保持表达能力的同时显著减少参数。

## 动态计算 (Dynamic Computation) ⚡

![](images/4f7106f307c7.png)

动态计算的目标与前几种方法不同：它并非单纯追求网络变小，而是希望**同一个网络能根据可用计算资源动态调整其计算量**。

![](images/da656a16d91b.png)

![](images/69e18ed0178b.png)

**应用场景**：

1. 同一模型需部署在不同算力的设备上。
2. 同一设备在不同状态下（如电量充足 vs 电量不足）对算力的需求不同。

![](images/a1fdac6576a8.png)

**为什么不直接准备多个不同大小的网络？** 因为存储多个网络模型会占用大量空间。我们期望一个模型就能灵活适配。

![](images/523a47f3ceb3.png)

### 实现动态计算的方法

**1. 动态深度 (Dynamic Depth)**

- 训练一个很深的网络，并在中间层插入额外的“提前退出分支”。
- 每个分支都可以输出分类结果。
- 训练时，要求所有分支的输出都尽量接近真实标签。
- 推理时，算力充足则使用最终输出；算力不足则可在浅层分支提前输出结果。

![](images/9819635d8963.png)

**2. 动态宽度 (Dynamic Width)**

- 训练时，让网络同时以多种“宽度”（即激活不同比例的神经元子集）进行前向传播。
- 要求所有宽度下的输出都接近真实标签。
- 推理时，根据可用算力选择对应的宽度。

**3. 网络自适应决策**

- 让网络根据输入样本的“难度”自行决定需要多少计算量。对于简单样本，早期层就能做出准确判断并提前退出；对于复杂样本，则运行到深层。
- 这可以进一步节省计算资源。

![](images/28b25cde7522.png)

## 总结 🎯

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/6f265fb5b21f41a90e902fa589cd2483_35.png)

本节课我们一起学习了多种神经网络压缩与加速技术：

1. **知识蒸馏**：让小型学生网络学习大型教师网络的输出分布，以提升小模型性能。
2. **参数量化**：通过降低数值精度、权值聚类等方法，减少存储每个参数所需的比特数。
3. **网络架构设计**：重点介绍了**深度可分离卷积**，通过将标准卷积分解为深度卷积和逐点卷积，大幅减少参数量。
4. **动态计算**：使网络能够根据可用计算资源或输入难度，动态调整其深度或宽度。

![](images/f422a69c9cfd.png)

需要注意的是，这些技术**并非互斥**。在实际应用中，为了达到极致的压缩效果，可以组合使用多种技术，例如先进行知识蒸馏，再进行网络剪枝，最后对权重进行量化。

### 在线策略 vs. 离线策略

![](images/3eab69b8325f.png)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/db6658133db27bd03532a790377cbc37_37.png)

![](images/20ec44611a89.png)

我们知道，在强化学习中，我们要训练的是一个智能体。

![](images/c4ee24af6b82.png)

![](images/d3b168dec09a.png)

如果我们用来与环境互动的智能体，和我们要学习的智能体是同一个，这叫做**在线策略**。如果我们学习的智能体，和与环境互动的智能体不是同一个，这叫做**离线策略**。

![](images/4def228a1cb2.png)

![](images/b62bbc808b19.png)

拟人化地说，如果智能体一边与环境互动一边学习，这叫在线策略。如果它是在旁边看别人玩，通过观察来学习，这叫离线策略。

![](images/639df77d9aec.png)

以下是棋魂的比喻：

- 如果阿光自己下棋自己学，这是在线策略。
- 如果他在旁边看佐为下棋，这是离线策略。

### 为何需要离线策略？

![](images/384a88630deb.png)

![](images/2d7c12c4cd0b.png)

![](images/86ac848b703d.png)

让我们回想一下之前讲过的策略梯度。我们之前讲的策略梯度，是在线策略还是离线策略的做法呢？

![](images/b375aeabce7a.png)

![](images/057e28b2bddb.png)

![](images/d8cbdd1a9c99.png)

它是**在线策略**的做法。

![](images/a071794c4747.png)

![](images/88cd3d1452cb.png)

为什么？我们之前讲在做策略梯度时，需要一个智能体，需要一个策略，需要一个执行者。这个执行者先去与环境互动，搜集资料，收集很多轨迹。根据收集到的资料，按照策略梯度的公式去更新策略的参数。这就是我们之前讲过的策略梯度。所以它是一个在线策略的算法，拿去与环境互动的策略和要学习的策略是同一个。

![](images/90bccc4371fe.png)

今天的问题是，在更新的公式里，其中有一项期望值，是针对你当前策略参数θ所采样出的轨迹τ取的期望。所以当你更新参数后，一旦参数从θ变成θ‘，这个概率就不对了，之前采样出的数据就变得不能用了。

![](images/e69d02d44550.png)

![](images/5dd6b0758955.png)

![](images/8da8b510b19d.png)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/db6658133db27bd03532a790377cbc37_75.png)

所以我们之前就讲过，策略梯度是一个会花很多时间来采样数据的算法。你会发现大多数时间都在采样数据。与环境互动后，接下来就要更新参数。每次更新完参数一次，你只能做一次梯度下降。更新完参数一次，接下来你就要重新再去收集数据，然后才能再次更新参数。这显然是非常花时间的。

### 离线策略的优势

![](images/a72e3db8b2e5.png)

现在我们想从在线策略变成离线策略的好处是，我们希望可以用另外一个策略，另外一个执行者θ‘去跟环境做互动，用θ‘收集到的数据去训练θ。

![](images/71c5d0a9ddfc.png)

![](images/7c990daf501e.png)

![](images/efba7d0397a2.png)

假设我们可以用θ‘收集到的数据去训练θ，意味着我们可以把θ‘收集到的数据使用非常多次。在做梯度上升时，我们可以执行梯度上升好几次，可以更新参数好几次，都只要用同一批数据就好了。因为假设现在θ有能力从另外一个执行者θ‘所采样的数据来学习的话，那θ‘就只要采样一次，采样多一点的数据，让θ去更新很多次，这样就会比较有效率。

## 重要性采样

![](images/34a00a122f8d.png)

![](images/8857ba3d9a10.png)

![](images/f0e462d3d993.png)

那么怎么做呢？这边就需要介绍一个叫做**重要性采样**的概念。这个重要性采样的概念不是只能用在强化学习上，它是一个通用的想法，可以用在很多其他地方。我们先介绍这个通用的想法。

![](images/a19b97b0ff88.png)

假设现在你有一个函数f(x)，你要计算从分布P采样x，再把x带到f里面得到f(x)，你要计算这个f(x)的期望值。怎么做呢？假设你今天无法对分布P做积分，那你可以从分布P去采样一些xi，那么这个期望值，就近似于你采样到的xi，把xi带到f(xi)里面，然后取它的平均值。假设你知道怎么从分布P做采样的话，你要算这个期望值，你只需要从分布P做采样就好。

![](images/4d1f14782a93.png)

但是我们现在有另外一个问题。我们**无法**从分布P里面采样数据。我们只能从另外一个分布Q去采样数据。Q这个分布可以是任何分布。我们不能从P采样数据，但我们可以从Q去采样x^i。

![](images/adeb2f4819c1.png)

![](images/908dba9f9db5.png)

我们不能够直接套用从P采样的近似公式，因为那个公式假设你的xi都是从P采样出来的。从Q采样出来的xi套那个公式，你不会等于左边这一项期望值。

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/db6658133db27bd03532a790377cbc37_103.png)

所以怎么办？作为一个修正，这个修正是这样的。期望值这一项，其实就是积分f(x)乘上P(x)。我们现在上下都同乘Q(x)，上下同乘Q(x)不会改变任何事。但是我们可以把这个式子写成，对Q里面所采样出来的x取期望值。

![](images/5f3f95c9466b.png)

![](images/5f3e7430c7cb.png)

我们从Q里面采样x，然后再去计算f(x)乘上P(x)除以Q(x)，再去取期望值。左边这一项会等于右边这一项。

![](images/33bce79d1c03.png)

要算左边这一项，你要从分布P采样x。但要算右边这一项，你不是从分布P去采样x，你是从分布Q去采样。你从分布Q采样出来以后，再代入f(x)乘上P(x)/Q(x)，接下来你就可以计算左边这项你想要算的期望值。

![](images/186674847648.png)

![](images/43af1fcbfeae.png)

所以就算是我们不能够从P里面采样数据，你想要计算这一项的期望值也是没有问题的。你只要能够从Q里面去采样数据，代入这个式子，你就一样可以计算从分布P采样x代入f以后所算出来的期望值。

![](images/f940c3e5d747.png)

![](images/2f7d731ab98d.png)

这两个式子唯一不同的地方是，一个是从P做采样，一个是从Q做采样。因为它是从Q里面做采样，所以采样出来的每一笔数据你需要乘上一个权重，修正这两个分布之间的差异。而这个权重就是P(x)的值除以Q(x)的值。

![](images/0d944475f4b9.png)

![](images/e80dbdaa5410.png)

![](images/9c37da47ebbb.png)

所以Q(x)其实它是任何分布都可以。这边唯一的限制就是你不能够在P(x)不为零但Q(x)为零的时候使用，不然会没有定义。假设Q(x)是零的时候，P(x)也都是零的话，那这样P/Q是有定义的，这个时候你就可以应用重要性采样这个技巧。所以你就可以从本来是从P做采样，换成从Q做采样。

### 重要性采样的问题

![](images/0b8455483d2d.png)

这个跟我们刚才讲的有什么关系呢？跟刚才讲的从在线策略变成离线策略有什么关系呢？在继续讲之前，我们来看一下重要性采样的**问题**。

![](images/024c2403c839.png)

![](images/0a2e13a09cc3.png)

![](images/5775528984f8.png)

在理论上你可以把P换成任何的Q，但是在实践上并没有那么容易。实践上P跟Q还是不能够差太多，如果差太多的话，会有一些问题。什么样的问题呢？

![](images/2fa9ab81de07.png)

虽然我们知道左边这个式子的期望值等于右边这个式子的期望值。左边这个随机变量的分布是P，右边这个随机变量（f(x) * P(x)/Q(x)）的分布是Q。我们现在如果不是算期望值，而是算方差的话，这两个方差你觉得他会一样吗？它们是不一样的。两个随机变量，它们的均值一样，并不代表它的方差一样。

![](images/c7b8ac092f92.png)

![](images/748e024d484d.png)

所以今天你可以实际算一下，f(x)这个随机变量跟f(x)乘以P/Q这个随机变量，它们的方差是不是一样的。计算后会发现，如果P除以Q差距很大的话，后面这一项的方差就会很大。

![](images/f2bed116b0a2.png)

![](images/ea61a03b05d0.png)

所以虽然理论上它的期望值一样，也就是说你只要对分布P采样足够多次，对分布Q采样足够多次，你得到的结果会是一样的。但是假设你采样的次数不够多，因为它们的方差差距是很大的，所以你就有可能得到非常大的差别。

![](images/ddc7aa30e852.png)

如果这个地方你听的不是很懂，那没有关系，这边举一个具体的例子，告诉你说当P跟Q的差距很大的时候，会发生什么样的问题。

![](images/ee3eadbc466f.png)

![](images/4500542f65e4.png)

假设P的分布集中在左侧，Q的分布集中在右侧，f(x)在左侧是负的，在右侧是正的。如果我们要计算f(x)的期望值，它的分布是从P的分布来采样，那显然这一项是负的。因为f(x)在P(x)几率高的区域是负的，只要采样都会采样到这个地方，所以理论上这一项算出来会是负的。

![](images/529c18c2733c.png)

![](images/21709315a2c7.png)

![](images/df773abbfd66.png)

接下来我们改成从Q这边做采样。因为Q在右边这边的几率比较高，所以如果你采样点不够的话，那你可能都只采样到右侧。如果你都只采样到右侧的话，你会发现算起来右边这一项搞不好是正的。因为你这边采样到这些点，然后你去计算它们的f(x) * P/Q都是正的，所以你采样这些点他们都是正的，所以你取期望值以后也都是正的。

![](images/efdd31e685ab.png)

但为什么会这样？那是因为你采样的次数不够多。因为假设你采样次数很少，有可能只采样到右边这边。但左边这边几率虽然很低，但也不是没有可能被采样到。假设你今天好不容易终于采样到左边的点，因为左边的点P跟Q是差很多的，Q很小，P很大，所以今天f(x)好不容易采样到一个负的，这个负的就会被乘上一个非常大的权重，就可以平衡掉刚才那边一直采样到正值的情况。最后终究还是可以算出这一项的期望值是负的。

![](images/bb67708eb2ab.png)

![](images/66b638e8271f.png)

但问题就是这个前提是你要采样足够多次，这件事情才会发生。但有可能采样次数不够，那左式和右式就有可能有很大的差距。

![](images/e70f6d979f9b.png)

![](images/032d255e78d7.png)

所以这是重要性采样的问题。

![](images/e3639881a1de.png)

![](images/a1c1ed182c49.png)

![](images/08e46d0fe37b.png)

![](images/83a7717c43d3.png)

## 将重要性采样用于离线策略

![](images/8652b376cf83.png)

![](images/8f593b763674.png)

![](images/ec5d0404c6ff.png)

现在要做的事情就是把重要性采样这件事，用在离线策略的情况。我要把在线策略的训练算法，改成离线策略的训练算法。

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/db6658133db27bd03532a790377cbc37_185.png)

![](images/839e2fcf6712.png)

![](images/2751193105da.png)

怎么改呢？之前我们是拿θ这个策略去跟环境互动，采样出轨迹τ，然后计算目标函数。现在呢，我们不根据θ，我们不用θ去跟环境做互动。我们假设有另外一个策略，它的参数是θ‘，它就是另外一个执行者。他的工作是他要去做示范，他要去示范给你看。这个θ‘，他的工作是要去示范给θ看，他去跟环境做互动，告诉θ说他跟环境做互动时会发生什么事，然后借此来训练θ。我们要训练的是θ这个模型，θ‘只是负责做示范的，负责跟环境做互动。

![](images/b420f9c0711b.png)

![](images/2b14b9371241.png)

![](images/e0ca38bcd4f3.png)

所以我们现在采样出的τ是从θ‘采样出来的，不是从θ采样出来。但我们本来要求的目标函数是针对θ的期望。我们实际上做的时候是拿θ‘去跟环境做互动，所以采样出来τ是从θ‘采样出来的。这两个分布不一样，但没有关系。我们之前说过，假设你本来是从P做采样，但你发现你不能够从P做采样（就现在我们说我们不拿θ去跟环境做互动，所以不能从P做采样），你永远可以把P换成另外一个Q，然后在后面这边补上一个重要性权重。

![](images/79134676caa7.png)

![](images/d37ab1e9ce69.png)

![](images/e2f7a764e95e.png)

所以现在的状况就是一样，把θ换成θ‘以后，要在目标函数里面补上一个重要性权重。这重要性权重就是某一个轨迹τ，它用θ算出来的几率除以这个轨迹τ，它用θ‘算出来的几率。

![](images/2210f75183bd.png)

![](images/5776d2df6b29.png)

这一项是很重要的，因为今天你要学习的执行者θ跟θ‘是不太一样的，所以θ‘会遇到的状况、会见到的情形跟θ见到的情形不见得是一样的，所以中间要做一个修正的项。

![](images/99510b4c4a35.png)

![](images/3e175de74494.png)

![](images/4516db8e8549.png)

所以我们做了一下修正，因为我们现在数据不是从θ采样出来的，是从θ‘采样出来的。

![](images/7ca5afe0bd82.png)

![](images/5e54a4b6bafd.png)

![](images/f16fddcd0c2e.png)

### 离线策略的好处

![](images/cac7c29e6097.png)

![](images/07f6a43b082a.png)

![](images/fc2cfa9bafeb.png)

我们从θ换成θ‘有什么好处呢？我们刚才就讲过，因为现在跟环境互动的是θ‘，而不是θ，所以今天你采样出来的东西跟θ本身是没有关系的。所以你就可以让θ‘去跟环境做几次互动，采样一大堆数据以后，θ可以更新参数很多次，然后一直到θ可能训练到一定的程度，它被更新很多次以后，θ‘再重新去做采样。这就是在线策略换成离线策略的妙用。

### 具体的目标函数

![](images/5a23fb4ddb95.png)

![](images/183d0ee73028.png)

![](images/d797aa6666ea.png)

我们其实上周有讲过，实际上在做策略梯度的时候，我们并不是给一整个轨迹τ都一样的分数，而是每一个状态-动作对我们会分开来计算。所以我们上周其实都讲过，我们实际上更新梯度的时候，我们的式子是长这个样子的：

![](images/fbc9e229e376.png)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/db6658133db27bd03532a790377cbc37_239.png)

![](images/aa94123a9f13.png)

我们用θ这个执行者去采样出s_t跟a_t，采样出这个状态和动作的对。我们会计算这个状态-动作对的优势函数A^θ(s_t, a_t)，估测它有多好。这一项就是那个累计的奖励减掉基线。接下来后面会乘上log p_θ(a_t | s_t)。也就是说如果这一项是正的，就要增加这个动作的几率，这一项是负的，就要减少几率。

![](images/681753a138f9.png)

![](images/4e079a40848e.png)

那我们现在就用了重要性采样的技术，把在线策略变成离线策略。我就把在线策略变成离线策略，就从θ变成θ‘。所以现在s_t, a_t它不是θ跟环境互动以后所采样到的数据，它是θ‘另外一个执行者跟环境互动以后所采样到的数据，但是拿来训练我们要调整参数的模型θ。

![](images/caa51a0bd83c.png)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/db6658133db27bd03532a790377cbc37_249.png)

![](images/6563876c792b.png)

但是我们有说过，因为θ‘跟θ是不同的模型，所以你要做一个修正的项。而这项修正项就是重要性采样的技术，把s_t, a_t用θ采样出来的几率除掉s_t, a_t用θ‘采样出来的几率。

![](images/3c2aa8a4d65c.png)

![](images/e4ef0d847d38.png)

![](images/a7e472cadeb5.png)

这边其实有一件事情我们需要稍微注意一下。这边A有一个上标θ，代表这个是执行者θ跟环境互动的时候所计算出来的A。但是实际上，当我们今天从θ换到θ‘的时候，这一项其实你应该改成A{θ‘}，而不是Aθ。因为这个优势函数A这一项是怎么来的？这一项是想要估测说现在在某一个状态，采取某一个动作，接下来会得到的累计奖励的值减掉基线。之前是θ在跟环境做互动，所以你观察到的是θ可以得到的奖励。但现在不是θ跟环境做互动，现在是θ‘在跟环境做互动，所以你得到的这个优势函数，其实是根据θ‘所估计出来的优势函数。但我们现在先不要管那么多，我们就假设这两项可能是差不多的。

![](images/24d3f0a2074e.png)

![](images/790c2752d105.png)

接下来呢，这个p_θ(s_t, a_t)这一件事情你可以拆解成p_θ(s_t)的几率乘上p_θ(a_t|s_t)的几率。然后接下来这边需要做一件事情，是我们假设当你的模型是θ的时候，你看到s_t的几率跟你的模型是θ‘的时候，你看到s_t的几率是差不多的，你把它删掉，因为它们是一样的。为什么可以假设它是差不多的？当然你可以找一些理由，举例来说，会看到什么状态往往跟你采取什么样的动作是没什么太大的关系的。但是有一个更直接的理由就是，这一项到时候真的要你算，你会算吗？你不觉得这一项其实你不太能算吗？尤其是你如果输入是图像的话，同样的s_t根本就没有出现过第二次，所以根本没办法估这一项。所以干脆就无视这个问题。而p_θ(a_t|s_t)这个几率你是会算的，对不对？这个很好算，因为你有θ这个参数，它就是个网络，你就把s_t带进去，它就会告诉你某一个动作的几率是多少。

![](images/66f2f51cd177.png)

![](images/0b5c9035f5c3.png)

![](images/3f6586fe81d0.png)

![](images/8d447e8759b8.png)

所以这一项你只要知道θ参数，知道θ‘参数，这个就可以算。那p(s_t)这一项不太好算，这就说服自己说这一项其实不太会有影响，我们只管前面这个部分就好了。

![](images/3968de7d3d03.png)

## 近端策略优化 (PPO)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/db6658133db27bd03532a790377cbc37_303.png)

但是我们会遇到的问题是，我们在前面讲重要性采样的时候，我们说重要性采样有一个问题：其实你的p_θ跟p_{θ‘}不能差太多。差太多的话，你重要性采样的结果就会不好。如果p_θ跟p_{θ‘}差太多的话，重要性采样的结果就会不好。

![](images/d79c38862616.png)

![](images/c205bfdccd6e.png)

所以怎么避免它差太多呢？这个就是PPO在做的事情。

![](images/6d15da549877.png)

![](images/7709ebc92e65.png)

![](images/817c5c1cec0f.png)

### PPO 的核心思想

![](images/6e323de454ae.png)

![](images/0135caef4b37.png)

PPO虽然如果你看它原始的论文，它里面写了很多的数学式，但它实际上做的事情是怎样呢？它实际上做的事情就是这样，它说我们原来在离线策略的方法里面，我们要优化的是这个目标函数。但是我们又说，因为这个目标函数它牵涉到重要性采样，而重要性采样要求p_θ不能跟p_{θ‘}差太多，你做示范的模型跟那个真正的模型不能够差太多，差太多的话，重要性采样的结果就会不好。

![](images/f7a0a54f0e95.png)

![](images/967d57b68fcb.png)

![](images/7227ecffefad.png)

所以我们在训练的时候**多加一个约束**。这个约束是什么？这个约束是θ跟θ‘这两个模型，它们输出的动作分布的KL散度。简单来说这一项的意思，就是要衡量说θ跟θ‘有多像。然后我们希望在训练的过程中，我们训练出来的θ跟θ‘越像越好，因为θ如果跟θ‘不像的话，最后你做出来的结果就会不好。

![](images/684d4a66065b.png)

![](images/bd149735ab5c.png)

![](images/ff9613b5f491.png)

所以在PPO里面有两个式子，一方面就是优化你本来要优化的东西，但是再加一个约束，这个约束就好像那个正则化项一样。我们在做训练的时候，不是有L1跟L2正则化吗？这一项也很像正则化项。正则化做的事情，就是希望最后训练出来的θ不要跟θ‘太不一样。

![](images/26762594753b.png)

![](images/0371bdf9f80a.png)

![](images/3e1cb7c79a64.png)

![](images/50040102db75.png)

### PPO 与 TRPO

![](images/8f190f583911.png)

![](images/112a5ac9e16d.png)

![](images/1bf84670f2a7.png)

PPO有一个前身叫做TRPO。TRPO写的式子呢是这个样子的，它唯一不一样的地方是说，这个约束摆的位置不一样。PPO是直接把那个约束放到你要优化的那个式子里面，然后接下来你就可以用梯度上升的方法去最大化这个式子。但是如果是在TRPO的话，它是把KL散度当做一个约束，希望θ跟θ‘的KL散度小于一个δ。你知道在做那种有约束的优化时，如果你用基于梯度下降的优化方法，有约束是很难处理的。所以你会发现PPO在实践上比TRPO容易得多。

![](images/429c8b06aa52.png)

![](images/315ddfa77df6.png)

文献上的结果是，PPO跟TRPO可能表现差不多，但是PPO在实践上比TRPO容易得多。

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/db6658133db27bd03532a790377cbc37_349.png)

![](images/807d8fd76e52.png)

### 关于KL散度的注意点

![](images/6274e00beb20.png)

![](images/cc53bae2637e.png)

![](images/afc0a9f79921.png)

这边要注意一下，就是这边所谓的KL散度到底指的是什么？这边我是直接把KL散度当做一个函数，它吃的输入是θ跟θ‘。但我的意思并不是说把θ当做一个分布，把θ‘当做一个分布，算这两个分布之间的距离。我不是这个意思。
