## 背景知识

![](images/0ef474a75468.png)

在深入探讨新的优化器之前，我们需要了解一些基础概念和术语。这些概念是理解后续内容的关键。

**核心符号定义：**

- **θ**：代表整个神经网络的参数，即我们训练中想要优化的目标。
- **L(θ)**：表示在特定参数θ下的损失函数值。
- **∇L(θ)**：损失函数L关于参数θ的梯度。
- **m**：动量（Momentum），它是一个向量，记录了历史梯度信息的加权和。

![](images/e9361d1bf8f5.png)

**神经网络训练流程：**  

在每个时间步t，我们有一个输入 **x_t**，通过当前参数为 **θ_t** 的网络，得到预测输出 **ŷ_t**。将 **ŷ_t** 与真实标签 **y_t** 进行比较，计算出损失 **L(θ_t)**。优化器的目标就是找到一组参数 **θ**，使得在所有训练数据上的总损失最小。

![](images/7479df5bb2d5.png)

## 优化目标与设定

![](images/ec85d602e0a4.png)

上一节我们介绍了训练的基本流程，本节中我们来看看优化的具体目标。

我们的目标是找到一组参数 **θ***，使得在训练集上所有样本的损失之和最小。用公式表示如下：

**θ* = argmin_θ Σ L(θ; x, y)**

![](images/919275bf6cbc.png)

这相当于在一个由损失函数构成的“曲面”（Loss Surface）上寻找最低点。然而，在实际训练中，我们通常无法一次性看到所有数据（离线学习）。更常见的做法是每次只使用一个或一小批数据（在线学习）。为了简化讨论，本课程将主要关注**离线优化**的设定，即假设在每一步都能计算基于全部数据的完整梯度。

## 经典优化算法回顾

![](images/f4c8809fcf65.png)

在探索新方法之前，我们先快速回顾几种经典的优化算法。以下是五种最常用的优化器：

![](images/a8f6b890c05f.png)

![](images/367c97ef1a56.png)

1. **SGD (随机梯度下降)**
  
  **公式**：`θ_{t+1} = θ_t - η * ∇L(θ_t)`
  **描述**：最基础的优化方法。直接沿着当前点的负梯度方向更新参数。学习率η是固定的。

![](images/f5dd51c9f65c.png)

1. **SGDM (带动量的SGD)**
  
  **公式**：
  
  `m_{t+1} = β * m_t + (1-β) * ∇L(θ_t)`
  `θ_{t+1} = θ_t - η * m_{t+1}`
  
  
  **描述**：引入动量项`m`，它累积了历史梯度的信息。当前更新方向不仅取决于当前梯度，还受到历史更新方向的影响。这有助于在梯度很小（如平原区或局部最优点附近）时保持前进的惯性，可能帮助跳出局部最优。
2. **Adagrad**
  
  **公式**：
  
  `v_{t+1} = v_t + (∇L(θ_t))^2`
  `θ_{t+1} = θ_t - η / (√v_{t+1} + ε) * ∇L(θ_t)`
  
  
  **描述**：自适应学习率方法。分母`v`累积了历史梯度的平方和。对于历史上梯度较大的参数，其学习率会变小（更新更谨慎）；对于历史上梯度较小的参数，其学习率相对较大（更新更快）。但`v`会单调递增，可能导致后期学习率过小，训练提前停止。
3. **RMSProp**
  
  **公式**：
  
  `v_{t+1} = α * v_t + (1-α) * (∇L(θ_t))^2`
  `θ_{t+1} = θ_t - η / (√v_{t+1} + ε) * ∇L(θ_t)`
  
  
  **描述**：针对Adagrad的改进。分母`v`的计算引入了衰减因子α（通常接近1，如0.9），使得历史信息的影响呈指数衰减，而不是无限累积。这避免了学习率过早衰减的问题。
4. **Adam**
  
  **公式**：
  
  `m_{t+1} = β1 * m_t + (1-β1) * ∇L(θ_t)`
  `v_{t+1} = β2 * v_t + (1-β2) * (∇L(θ_t))^2`
  `m̂_{t+1} = m_{t+1} / (1 - β1^{t+1})`
  `v̂_{t+1} = v_{t+1} / (1 - β2^{t+1})`
  `θ_{t+1} = θ_t - η / (√v̂_{t+1} + ε) * m̂_{t+1}`
  
  
  **描述**：结合了动量（SGDM）和自适应学习率（RMSProp）的思想。`m`估计梯度的一阶矩（均值），`v`估计梯度的二阶矩（未中心化的方差）。`m̂`和`v̂`是对初始偏置的校正。Adam因其优秀的性能成为目前最流行的优化器之一。

## 现状：为何新优化器难以超越Adam/SGDM？

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/fe7fb0d1f872db6204504f0d37e5b856_35.png)

![](images/ce63748a2344.png)

![](images/ed1ba4c2d670.png)

回顾了经典算法后，我们不禁要问：自2014-2015年Adam提出后，为什么后续鲜有能广泛取代它的新优化器出现？许多知名模型依然主要使用Adam或SGDM。

![](images/da5940f25aef.png)

![](images/1765634c78dd.png)

![](images/813b58a281cf.png)

**以下是一些知名模型及其使用的优化器：**

- **BERT, Transformer, WaveNet, BigGAN, MAML**：使用 **Adam** 训练。
- **YOLO, Mask R-CNN, ResNet**：使用 **SGDM** 训练。

![](images/9d828a9ffb76.png)

**观察到的现象：**  

通过对比实验（如图像分类、语言模型训练）可以发现：

1. **Adam** 通常在训练初期收敛速度非常快，训练损失下降迅速。
2. **SGDM** 虽然初期可能较慢，但往往更稳定，最终收敛到的解在验证集上泛化性能可能更好，且训练曲线更平滑。

![](images/6c04fc4c7358.png)

**原因分析：**  

Adam和SGDM似乎占据了两个“极端”的优势位置：Adam代表“快速收敛”，SGDM代表“稳定泛化”。后续的许多改进尝试只是在两者之间寻找平衡点，但难以在综合性能上显著超越这两个基准。因此，在实践中，研究者往往直接在这两者中选择，而较少采用更新的、但提升不明显的优化器。

## 改进思路一：结合Adam与SGDM (SWATS)

既然Adam快而SGDM稳，一个自然的想法是结合两者。SWATS (2017) 正是这样的尝试。

**核心思想**：在训练初期使用Adam，利用其快速收敛的特性；在训练后期切换到SGDM，利用其良好的收敛性和泛化能力。

![](images/0e1fa9054f25.png)

**挑战**：

1. **切换时机**：何时从Adam切换到SGDM？
2. **学习率转换**：Adam的自适应学习率与SGDM的固定学习率如何衔接？

SWATS提供了一种启发式的切换方法和学习率初始化方案，但它更像一个工程性的“技巧”，缺乏严格的理论指导，且切换点需要根据经验设定。

## 改进思路二：修正Adam的问题

![](images/406e29b6b50e.png)

另一种思路是直接改进Adam，让它能像SGDM一样稳定收敛。这需要先分析Adam存在的问题。

![](images/98dfef60281e.png)

**问题分析：自适应学习率的副作用**  

考虑Adam中自适应学习率的分母项`v_t`。它是对梯度平方的指数移动平均。在训练后期，大部分梯度很小，偶尔会出现一个非常大的梯度（提供重要方向信息）。这会导致：

- 当大梯度出现时，由于`v_t`瞬间变得很大，其对应的有效更新步长会被压缩得非常小。
- 而之前那些由小梯度主导的步骤，由于`v_t`较小，反而进行了相对较大的更新。

![](images/004cc8e8b1f1.png)

这意味着，真正提供重要信息的梯度对参数更新的贡献被削弱了，而噪声较多的梯度却主导了更新，这可能损害收敛的最终质量。

![](images/bae652fd0ff0.png)

**解决方案尝试：AMSGrad (2018)**  

AMSGrad试图解决上述问题。它修改了`v_t`的更新规则，不再使用简单的指数平均，而是保留历史最大值：

![](images/82e317f3566b.png)

`v_{t+1} = max(β2 * v_t, (∇L(θ_t))^2)`  

`θ_{t+1} = θ_t - η / (√v_{t+1} + ε) * m̂_{t+1}`

![](images/1b7499f9faa6.png)

这样做的目的是防止`v_t`在遇到大梯度后，又因后续的小梯度而迅速变小，从而确保大梯度信息能持续影响后续的学习率，避免其更新步长被过度压缩。然而，这种方法可能重新引入类似Adagrad中学习率单调下降的问题。

![](images/9ff953e27c66.png)

![](images/7c2590457c90.png)

**解决方案尝试：AdaBound (2019)**  

AdaBound从另一个角度出发，直接对Adam的学习率进行裁剪（Clipping），为其设置一个动态变化的上下界。

![](images/1afbb9c7f01e.png)

![](images/44d2b60a65a2.png)

`lr_t = clip(η / √v̂_t, lower_bound(t), upper_bound(t))`  

`θ_{t+1} = θ_t - lr_t * m̂_{t+1}`

![](images/47db5baceb40.png)

![](images/1d54cb7a68d2.png)

![](images/b22a8e11724b.png)

上下界是训练步数`t`的函数，初期较宽，允许自适应学习率发挥作用；后期逐渐收紧，最终将学习率限制在一个固定小区间内，使其行为接近SGDM。这个方法简单有效，但引入的人工裁剪边界显得有些“粗暴”，失去了完全自适应的优雅性。

![](images/4ca15754e21e.png)

## 改进思路三：加速SGDM

我们也可以从另一个方向思考：如何让SGDM变得像Adam一样快？关键在于学习率的调整。

![](images/8fc82fc103aa.png)

![](images/b77d6be3c601.png)

**学习率范围测试 (LR Range Test)**  

这是一个实用的技巧：以不同的学习率训练模型几个周期，观察训练损失的变化。通常会找到一个“最佳区间”，学习率太小或太大效果都不好。

![](images/2aba8f99ea88.png)

**周期性学习率 (Cyclical LR) 与单周期学习率 (One-Cycle LR)**  

这些方法不再使用单调下降的学习率调度，而是让学习率在训练过程中周期性变化。

- **核心思想**：学习率较大时进行“探索”，可能帮助跳出局部最优或平坦区域；学习率较小时进行“利用”，在最优解附近精细收敛。
- **Cyclical LR**：学习率在预设的上下界之间以三角波等形式循环变化。
- **One-Cycle LR**：只进行一次“上升-下降”循环：先从一个较小的学习率“热身”上升至最大值，然后缓慢下降至远低于初始值，最后再有一个极小的“微调”阶段。

![](images/d72ea84cab80.png)

这些方法通过精心设计的学习率调度策略，在一定程度上提升了SGDM的收敛速度。

## 总结

本节课我们一起学习了深度学习优化器的发展脉络：

1. 我们首先回顾了**SGD, SGDM, Adagrad, RMSProp, Adam** 这五大经典优化器，理解了它们的基本原理和公式。
2. 我们探讨了当前优化器领域的现状：**Adam** 和 **SGDM** 因其分别在“快速收敛”和“稳定泛化”上的优势，成为难以被全面超越的基准。
3. 我们分析了三种主要的改进思路：
  
  **结合派**：如SWATS，尝试在训练的不同阶段分别使用Adam和SGDM。
  **修正派**：如AMSGrad和AdaBound，旨在解决Adam在训练后期因自适应学习率可能产生的问题，使其收敛更稳定。
  **加速派**：如周期性学习率调度，通过动态调整SGDM的学习率来提升其收敛速度。

![](images/72edd06b171b.png)

目前尚无一个“完美”的优化器能适用于所有任务。在实践中，选择Adam（追求快速实验迭代）或SGDM（追求最佳泛化性能）并搭配合适的学习率调度策略，仍然是可靠的选择。理解这些优化器的特性，有助于我们根据具体任务做出更明智的决策。
