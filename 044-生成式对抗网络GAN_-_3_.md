## 1. GAN的训练难点 🤔

![](images/62f6e0112800.png)

上一节我们介绍了WGAN及其改进。然而，即便有了WGAN，GAN的训练仍然以困难著称。其本质困难在于判别器（Discriminator）和生成器（Generator）的相互依赖关系。

- **判别器（D）的目标**：区分真实图片和生成器产生的假图片。
- **生成器（G）的目标**：生成足以骗过判别器的假图片。

![](images/7945f4bf8f53.png)

这两个网络在训练中相互砥砺、共同成长。一旦其中一方训练出现问题而停止进步，另一方也会随之停滞。

![](images/91fd78ea090c.png)

![](images/97b155a72f9a.png)

![](images/4fbd99535cd3.png)

- 如果判别器训练不佳，无法有效区分真假，生成器就失去了明确的优化目标，无法进步。
- 如果生成器无法进步，不能产生更逼真的图片，判别器也就无法进一步学习，同样会停滞。

![](images/cd4c0d19145a.png)

训练GAN时，双方必须“棋逢敌手”。任何一方“放弃比赛”，整个训练过程就可能崩溃。由于训练过程是自动的，我们无法在每次迭代中都手动调整超参数，因此祈祷每次训练判别器时损失都能稳定下降。一旦某次训练失败，整个过程就可能失败。

![](images/b1afa9c1f12b.png)

![](images/e0f9ca28d212.png)

![](images/19f56acd5ef3.png)

![](images/a09df7cf4cdd.png)

![](images/4eb20401da76.png)

![](images/7d2341553b21.png)

![](images/41bf0a984466.png)

![](images/c08bbee56056.png)

**核心互动关系**可以概括为一种对抗性优化：

```
min_G max_D V(D, G) = E_{x~p_data(x)}[log D(x)] + E_{z~p_z(z)}[log(1 - D(G(z)))]
```

其中，`D`试图最大化此目标以更好地区分，而`G`试图最小化此目标以更好地生成。

## 2. GAN在文本生成上的特殊挑战 📝

![](images/99b143bf917b.png)

![](images/665dcfd99479.png)

![](images/8aae468b2cf9.png)

![](images/a5a472ae05b1.png)

![](images/15f769bacd4a.png)

![](images/72ed441b118a.png)

用GAN生成一段文字被认为是极大的挑战。原因在于生成文字通常使用序列到序列（Sequence-to-Sequence）模型（如Decoder）作为生成器。

![](images/c1d1361032df.png)

![](images/81cbfc2a0ace.png)

![](images/a574c36dbd8c.png)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/8dfc29f291e690c3ef49e70e82b1aaf9_97.png)

![](images/c8f9bd2b0eb0.png)

![](images/58a614b08c35.png)

![](images/ab6760196cc4.png)

**难点**：当我们想通过梯度下降训练生成器（Decoder），使其能骗过判别器时，会遇到问题。因为Decoder参数的微小变化，可能导致其输出的token概率分布只有微小变动，而最终通过`argmax`选取的token很可能不变。对于判别器而言，输入没有变化，输出分数也就没有变化，导致梯度为零，无法进行有效的梯度下降。

![](images/5e057f04d2b9.png)

![](images/a03e29064e4f.png)

![](images/6b774b178c6a.png)

![](images/1c90c9bf6d21.png)

![](images/8cba3724f283.png)

**解决方案**：将问题转化为强化学习（Reinforcement Learning）问题。然而，强化学习本身也以难训练闻名，两者结合使得训练异常困难。很长一段时间内，无人能成功从头训练一个文本GAN。

![](images/173da6449bc8.png)

![](images/ddce3415d61e.png)

![](images/2dfb1d05f8bd.png)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/8dfc29f291e690c3ef49e70e82b1aaf9_121.png)

![](images/f039218db77b.png)

直到一篇名为“ScratchGAN”的论文出现，它通过**大量调整超参数**和集成多种技巧（如使用非常大的batch size、对判别器进行正则化、修改强化学习算法等），才实现了从零开始训练文本GAN。

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/8dfc29f291e690c3ef49e70e82b1aaf9_127.png)

## 3. 生成模型的比较与选择 ⚖️

![](images/d21b4234020f.png)

![](images/73aef081c138.png)

生成式模型不止有GAN，还有变分自编码器（VAE）和基于流的模型（Flow-based Model）。本课程选择重点讲解GAN，主要基于以下考虑：

![](images/867028eb916d.png)

![](images/a643c600ee58.png)

- **性能**：在生成高质量图片（尤其是复杂图片）方面，GAN通常优于VAE和Flow-based Model。
- **训练难度**：虽然GAN训练困难，但VAE和Flow-based Model同样不易训练。它们的损失函数包含多项，需要精细平衡才能得到好结果，难度不亚于GAN。

因此，本课程以实用为导向，选择介绍目前性能通常更优的GAN。对于其他模型，有兴趣的同学可以自行深入研究。

## 4. 生成器的评估方法 📊

![](images/6fae390136c3.png)

评估一个生成器的好坏并非易事。早期研究甚至完全依赖人眼主观判断。我们需要更客观、自动化的评估方法。

![](images/82224a7c822f.png)

### 针对特定任务的方法

例如，在生成动画人物头像的任务中，可以使用一个动画人脸检测系统，统计生成器产生的1000张图片中能检测到多少人脸。检测到的人脸越多，通常意味着生成质量越好。

![](images/24c2ba36b693.png)

### 通用评估方法：借助图像分类器

一个常见思路是使用预训练的图像分类器（如Inception Network）来评估生成图片。

![](images/a9f30ad875a3.png)

![](images/b84d6efdb408.png)

- **评估生成质量（Quality）**：将单张生成图片输入分类器，观察其输出的类别概率分布。分布越集中（分类器越确信图片属于某个类别），说明图片越清晰、逼真。
- **评估生成多样性（Diversity）**：将大量生成图片输入分类器，分别得到每张图片的概率分布，然后**计算所有这些分布的平均值**。这个平均分布越平坦（越均匀），说明生成器产生的图片类别越多样。

![](images/00d016cbbcdb.png)

**注意**：质量高要求单张图片的分布集中，多样性高要求所有图片平均分布平坦，两者评估角度不同。

![](images/434024081ac1.png)

### 常用评估指标

1. **初始分数（Inception Score, IS）**：同时衡量质量和多样性。分数越高越好。但其依赖的ImageNet分类器可能不适用于特定领域（如二次元头像）。
2. **Fréchet初始距离（Fréchet Inception Distance, FID）**：更常用的指标。方法如下：
  
  将真实图片和生成图片分别输入Inception Network，提取进入最终Softmax层之前的特征向量。
  假设这两组特征向量均服从多元高斯分布。
  计算这两个高斯分布之间的Fréchet距离（也称为Wasserstein-2距离）。距离越小，说明生成图片与真实图片在特征空间越接近，即生成质量越高且多样性越好。

**FID的局限性**：假设特征服从高斯分布可能不准确，且需要生成大量图片才能准确估计分布，计算量较大。在实践中，常结合多个指标（如FID和任务特定指标）进行综合评估。

![](images/a874704cff9a.png)

### GAN特有的评估难题

即使指标很好，生成器仍可能存在问题：

- **模式崩溃（Mode Collapse）**：生成器只产生少数几种样本，缺乏多样性。例如，生成的人脸总是同一张。这是判别器的“盲点”，生成器抓住后便停止创新。目前尚无根本解决方法，实践中常在崩溃前保存模型。
- **模式丢弃（Mode Dropping）**：生成器只能覆盖真实数据分布的一部分。例如，生成的人脸多样性看似足够，但全部是某种肤色。这个问题更隐蔽，更难通过现有指标侦测。
- **记忆与复制**：生成器可能只是“背诵”并复制训练集中的图片，而非学会泛化生成新图片。这也不是现有通用指标能完全发现的。

![](images/151d706f1ae4.png)

评估生成模型本身就是一个活跃的研究领域。

## 5. 条件生成式对抗网络（Conditional GAN）🎨

![](images/77e4175a51c9.png)

![](images/b431a8648053.png)

基本的GAN输入是随机噪声，输出不可控。条件GAN则允许我们控制生成过程，即根据给定的条件`x`（如一段文字、一张草图）和随机噪声`z`，生成对应的输出`y`。

![](images/aa544b599ca9.png)

### 文字生成图片（Text-to-Image）

- **输入**：一段文字描述（如“红头发，绿眼睛”）。
- **输出**：符合该描述的图片。每次输入相同的文字但不同的噪声`z`，会生成不同的符合描述的图片。
- **训练数据**：需要**成对的**（文字，图片）数据。

### 关键设计：判别器的输入

在条件GAN中，判别器`D`的输入必须同时包含**生成的图片`y`**和**条件`x`**。`D`的任务是判断“`y`是否真实，且`y`是否与`x`匹配”。

- 给`D`高分的情况：真实的图片`y_real` + 与之匹配的正确条件`x`。
- 给`D`低分的情况：
  
  生成器产生的图片`y_fake` + 对应的条件`x`。
  （重要！）真实图片`y_real` + **不匹配**的条件`x_wrong`（例如，一张猫的图片配文字“狗”）。加入这种“负样本”对训练判别器至关重要。

![](images/96eb59032906.png)

### 图片生成图片（Image-to-Image Translation / Pix2Pix）

条件`x`也可以是图片，应用广泛：

- 草图/设计图 → 真实图片
- 黑白图片 → 彩色图片
- 白天图片 → 夜晚图片
- 有雾图片 → 去雾图片

![](images/9b633d910732.png)

![](images/385eec75bfdc.png)

**为何需要GAN？** 如果只用监督学习（如L1/L2损失）训练生成器，模型倾向于输出所有可能结果的“平均值”，导致结果模糊。加入GAN的对抗损失，可以鼓励生成更清晰、更真实的图片。通常，结合**对抗损失**和**与目标图像的相似度损失**（如L1损失）能取得最佳效果。

![](images/0354756db8fc.png)

![](https://github.com/OpenDocCN/dsai-notes-zh/raw/master/docs/leemldl-2026/img/8dfc29f291e690c3ef49e70e82b1aaf9_201.png)

![](images/178500c96008.png)

![](images/94dbdbb31208.png)

### 其他有趣应用

- **声音生成图片**：输入一段音频（如狗叫、水流声），生成与之对应的场景图片。模型能从大量视频（包含画面和同步音频）中自动学习配对关系。
- **让静态画像动起来**：输入一张人物肖像画，生成该人物讲话、做表情的动态视频。

本节课我们一起深入探讨了GAN的核心挑战与高级应用。我们了解到：

1. GAN训练困难的根本原因在于生成器与判别器动态博弈、相互依赖的特性。
2. 评估生成模型需要综合使用多种指标（如FID），并警惕模式崩溃、模式丢弃等问题。
3. 条件GAN通过向判别器和生成器同时提供条件信息，实现了可控的内容生成，在文字转图像、图像转图像等领域有着强大而有趣的应用。

![](images/135d62c0afa5.png)

![](images/e890047d3025.png)

![](images/3b1cc409342a.png)

GAN是生成式模型中的重要代表，虽然训练有挑战，但其生成的质量和灵活性使其成为当前不可或缺的技术。
